Three Dynamics Equations
🔥 must memorise all 3

Each equation answers a different question. Prof: "Why three? Because each answers something different."

p(s', r | s, a)
Question answered:
"What is the chance that action a in state s lands me in s' AND gives me exactly reward r?"
This is the primitive — the most complete description. All other quantities are derived from it. It captures stochastic transitions AND stochastic rewards jointly.
Use when: you need the full joint probability of (next state, reward) together.
P(s' | s, a) = Σr p(s', r | s, a)
Question answered:
"What is the probability I end up in s', regardless of what reward I receive?"
Sum over ALL possible reward values to eliminate r — this is marginalizing. Capital P = lowercase p summed over r.
Warehouse robot example: Robot moves to "reach customer" — arrives fast (+10) or slowly (+5) but same next state. P(reached | s, move) sums both reward outcomes.
⚠️ Most common exam mistake: writing P(s'|s,a) = p(s', r|s,a) for ONE specific r. The marginal requires summing over ALL r values.
E[R | s, a, s'] = Σr r · p(s', r | s, a) / P(s' | s, a)
Question answered:
"On average, what reward do I get when I go from s via a and land in s'?"
Numerator = weighted sum of all rewards on this transition. Denominator = normalises (total probability of making this transition).
Used in: the Bellman equation to evaluate action choices — lets you compare which action gives the best average reward per transition.

Section 3

Returns & Discounting

Gt = Rt+1 + γGt+1 — the seed of everything in RL.

🔥 midsem hot
Why cumulative reward?
♟️
Move 12: sacrifice pawn
Immediate reward ≈ −1
···28 moves later···
♚
Move 40: checkmate!
Cumulative return = WIN
We need Gt — a number that captures all future consequences, weighted by how far away they are.
Four Return Formulas
🔥 know all 4
F1 Discounted Return ▼
Gt = Rt+1 + γRt+2 + γ²Rt+3 + ··· = Σk=0∞ γk Rt+k+1

Sum of all future rewards, each shrunk by γ raised to its distance. A reward two steps ahead counts for γ² of its face value.

⚠️ Off-by-one trap: Gt starts at Rt+1, NOT Rt. This is the most common exam mistake.
F2 Recursive Form ← seed of Bellman ▼
Gt = Rt+1 + γ Gt+1

The return is self-referential — each Gt contains Gt+1. Replace Gt+1 with the expected return from state St+1 → Bellman equation.

🌱 This is the seed of Bellman — memorise it.
F3 Constant Reward Shortcut ▼
If R = constant: Gt = R / (1 − γ)

Only applies when every reward from the starting point is identical. Geometric series sum. Example: R=1, γ=0.9 → Gt = 1/0.1 = 10.

⚠️ Common mistake: applying this to non-constant sequences. Only works on constant TAILS — split the sequence first.
F4 Convergence Bound ▼
|Gt| ≤ Rmax / (1 − γ)

If rewards never exceed Rmax, the return is always finite — no matter how long the sequence. Discounting "tames" infinite horizons.

Bellman Expectation Equation — 3-Step Derivation
🔥 must know
1
Expand Gt one level
vπ(s) = Eπ[ Rt+1 + γ Gt+1 | St=s ]
Using Gt = Rt+1 + γGt+1 (recursive form from Section 3)
2
Separate expectations (linearity)
= Eπ[Rt+1] + γ Eπ[vπ(St+1)]
Because E[Gt+1 | St+1] = vπ(St+1) by definition of vπ
3
Expand into explicit sums
vπ(s) = Σa π(a|s) Σs',r p(s',r|s,a) [r + γ vπ(s')]
← This is the Bellman Expectation Equation. Two sums, two sources of randomness.

Section 5

Optimality & Bellman Optimality

One symbol changes everything: Σaπ(a|s) → maxa. That gap separates prediction from control.

🔥 midsem hot
The ONE Change That Changes Everything
Bellman EXPECTATION
vπ(s) = Σa π(a|s) Σs',r p(…) [r + γvπ(s')]
Question: How good is policy π?
Math: Linear — matrix inversion works
Solves: Policy Evaluation
⇒
Σ→max
Bellman OPTIMALITY
v*(s) = maxa Σs',r p(…) [r + γv*(s')]
Question: What's the best possible?
Math: Non-linear — need iteration
Solves: Value Iteration, Control
⚠️ Why non-linear forces iteration: max(f+g) ≠ max(f) + max(g). Superposition fails. As value estimates change, which action wins the max can flip. No closed-form solution — must iterate.
Optimal Value Functions
v*(s)
v*(s) = maxπ vπ(s)

Highest possible expected return from state s, across ALL policies. One policy achieves this maximum everywhere simultaneously.

q*(s,a)
q*(s,a) = maxπ qπ(s,a)

Best expected return when you commit to action a from s, then follow optimal policy. Gives policy directly: π*(s) = argmaxa q*(s,a).

Key relation
v*(s) = maxa q*(s,a)

Optimal state-value = best action-value you can get by choosing the optimal action.

Both Bellman Optimality Equations
🔥 write from memory
v*(s) = maxa Σs',r p(s',r|s,a) [r + γ v*(s')]
maxa
Agent
Picks the best action — this is the decision. The max makes this optimality, not expectation.
Σs',rp
Environment
For the chosen action, nature randomly lands you in s'. Agent controls action; environment controls where.
γv*(s')
Bootstrap
v* appears on both sides — self-consistency condition (fixed point), not circular reasoning.
q*(s,a) = Σs',r p(s',r|s,a) [r + γ maxa' q*(s',a')]
Σs',rp
First
Environment averages first — action a is already committed. Nature picks next state s'.
maxa'
INSIDE Σ
Agent picks optimally at s' — AFTER the environment reveals s'. max must be INSIDE the sum. This is the most common exam mistake.
❌ WRONG: maxa' Σ ... — this puts max OUTSIDE. That means average first then pick best, which is wrong. Agent sees s' before picking.

Section 6

Dynamic Programming

Start with zeros. Apply Bellman. Repeat. Convergence guaranteed.

🔥 midsem hot
Prediction vs Control
Prediction (Evaluation)
Operator: Σaπ(a|s)×[…] (weighted average)
Algorithm: Iterative Policy Evaluation
Goal: compute vπ for a given π
Control (Improvement)
Operator: maxa×[…] (greedy max)
Algorithms: Value Iteration / Policy Iteration
Goal: find the optimal policy π*

PI alternates: fully evaluate current policy → improve greedily → repeat until stable.

In-Place vs Out-of-Place Updates
exam trap
Out-of-Place (Synchronous)
2 arrays: V_old and V_new. ALL updates this iteration use V_old. V(s') is always from previous iteration. Reliable, predictable convergence.
V_new(s) = maxa Σ p[r + γ V_old(s')]
In-Place (Asynchronous)
1 array: overwrite immediately. V(W) may use just-updated V(C) from same iteration. Often converges faster — uses freshest info. Order of updates matters.
V(s) = maxa Σ p[r + γ V(s')] ← overwrite now
Generalised Policy Iteration (GPI)

Almost all RL algorithms are instances of GPI: interleave some evaluation with some improvement.
