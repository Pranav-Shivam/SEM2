Reward vs Value Function

Intuition first

A reward is the slap or the high-five you get right now, the instant you act. A value function is the wise friend whispering "yeah that felt good, but where does it lead?" Reward is immediate and local; value is long-term and forward-looking. The whole reason value exists is that immediate reward is often useless for deciding.

Prof.'s chess example makes this unforgettable. In her reward design, win = +1, loss = −1, every other move = 0. So mid-game you are in state S1 and can play move A (→ S2) or move B (→ S3). Both give reward 0. The reward signal is flat — it cannot tell the two moves apart. You are blind.

Value functions break the tie by encoding the future. The value of S2 is 7; the value of S3 is 3. Why? Following A leads, after ~5 moves, to a checkmate on your opponent. Following B loses your queen after 3 moves. Fewer moves to a payoff (B) looks tempting, but value sees past that: losing the queen wrecks the game. So even though both immediate rewards are 0, value says play A.

"Reward is an immediate signal that you get. Value function is a long-term decision-making signal which will decide how good is it for you to be in a state and do a particular action… Value functions will capture future reward directions also." — Prof. Sangeetha — Transcript (≈ 23:41 – 24:00)

The two value functions

Prof. introduced both flavours (full math deferred to Session 3):

  • State value V(s) — how good is it to be in state s.
  • Action value Q(s,a) — how good is it to take action a while in state s.

Her key practical note: when you cannot model the environment (e.g. an autonomous vehicle — you can't write down clean transition dynamics), you skip trying to value states and work directly with Q(s,a), because Q is "directly evaluating your policy." This is the seed of model-free RL.

Formal definitions (research-enriched)

State value — expected discounted return Vπ(s) = 𝔼π[ Σt γt Rt | S0=s ]
Action value — expected return after taking a Qπ(s,a) = 𝔼π[ Σt γt Rt | S0=s, A0=a ]
Link under the optimal policy V*(s) = maxa Q*(s,a)

Symbols: γ = discount factor (0–1, how much future matters) · Rt = reward at step t · 𝔼π = expectation under policy π. In Session 2 we set γ aside; the bandit is the special case with one state and no future.

Why this matters

Every modern value-based algorithm — Q-learning, SARSA, DQN — is built on Q(s,a). A Deep Q-Network is literally a neural net that approximates this same Q. When AlphaGo evaluated board positions, it was estimating a value function exactly like the chess "7 vs 3". Reward-only agents (no value) are short-sighted and fail at any task with delayed payoff: chess, trading, robotics, dialogue.

Research Enhanced

Bellman's insight (1957), still the backbone in 2024–26 deep RL. Value functions are useful precisely because they obey a recursive consistency: V(s) = 𝔼[R + γV(s′)] (the Bellman equation, full treatment next session). This recursion is what lets DQN, and its modern descendants (e.g. distributional and double-Q variants used in current Atari/robotics benchmarks), bootstrap a long-horizon value from one-step samples. The chess "7" is a stand-in for this recursive lookahead. — Sutton & Barto, RL: An Introduction 2nd ed., Ch.3; Bellman 1957.

Worked examples

Q. In the chess example, why does the immediate reward fail to choose between move A and move B?

  1. Both A and B are interior moves → reward = 0 for each.
  2. Comparing 0 vs 0 gives no information.
The immediate rewards are identical (0 = 0), so reward alone cannot rank the moves. Only the value function — which encodes the future checkmate (A) vs lost queen (B) — separates them.
Key insight: reward discriminates only when outcomes differ now. Value is mandatory whenever payoffs are delayed.

Q. Given V(S2)=7 and V(S3)=3 in state S1, which move does a value-greedy agent take, and what is the danger of using only "moves-to-payoff"?

  1. Value-greedy picks the higher value → V(S2)=7 → move A.
  2. "Moves-to-payoff" would prefer B (3 moves < 5 moves) — but that metric ignores what happens (losing the queen).
Choose move A (value 7). Counting moves is a misleading shortcut; value already folds in the cost of losing the queen, so 3 > 7 is wrong as a decision rule — use the value, not the move count.
Key insight: a student in the lecture made exactly this "fewer moves is better" mistake — value already accounts for consequences, so trust the value number.

Q. The environment is unmodellable (autonomous driving). Explain why we prefer Q(s,a) over V(s), and what "model-free" buys us.

  1. To act from V(s) you must know transitions P(s′|s,a) to look one step ahead — that is a model.
  2. Q(s,a) bakes the action directly into the value, so the greedy policy is just argmaxa Q(s,a) — no model needed.
  3. For driving, dynamics (other cars, friction, sensor noise) can't be cleanly modelled, so Q learned from experience is the only practical route.
Q(s,a) lets you choose actions without environment dynamics → "model-free" RL. This is exactly why DQN (which outputs Q for each action) dominates settings where a model is unavailable.
Key insight: "model-free" = learn values/policy from raw interaction, never needing P(s′|s,a). Prof. tied this directly to the autonomous-vehicle case.
Exam Alert · closed book

Likely question: "Differentiate reward and value function with an example" — reproduce the chess case (win+1/loss−1/else 0; V(S2)=7 via future checkmate vs V(S3)=3 via lost queen).

Most common mistake: saying value = sum of rewards already received. Value = expected future return. Also: confusing "fewer moves to reward" with "higher value".

Exam-critical: the V(s) vs Q(s,a) distinction and the model-free justification for Q. Prof. explicitly deferred the formula to Session 3 but expects the intuition now.

Exploration vs Exploitation

Intuition first

This is the dilemma of RL. Exploitation = cash in on what you already know is good. Exploration = gamble on something unproven to learn more. Pure exploitation is a trap: you lock onto the first thing that looked good and never discover anything better. Pure exploration never settles and never collects the reward. Good agents do both, in balance.

Prof.'s restaurant story nails it: three new hotels open near you. You try all three; hotel 1 had great food day one. After that you go only to hotel 1 forever — that's pure greedy. But maybe hotels 2 and 3 were just having an off day (the regular chef was absent), or hotel 1's quality later drops. Because you never go back, you never find out. "Always sticking with a greedy choice will never give you the options to learn."

"How do we generally learn? In your Google Map, don't you look for other possible options, or you always go for only the route that Google Map tells? … You learn by mistakes. That leaving room for mistakes is your exploration." — Prof. Sangeetha — Transcript (≈ 1:58:40 – 1:59:00)

The tic-tac-toe value-table walkthrough

Prof. used tic-tac-toe (a zero-sum game: one wins +1, other −1, sum 0) to show how an agent acts on a value table.

Set up initial state values. A balanced opening position gets value 0.5 (both players ~50% to win). A position where player A has already won = 1; where B won = 0 (values are from A's perspective).
Each player keeps their own table. Value depends on whose turn and whose strategy, so in game-playing RL every player maintains a separate "record book" of state values.
Make moves to learn. From a given state B you have, say, 4 actions leading to states valued 0.3, 0.2, 0.7, 0.6.
Greedy move (exploit). Pick the action landing on the highest value → 0.7. "Greedy is nothing but, given the situation, I am choosing the most valued thing."
Explore. Later from state D (actions valued 0.2, 0.4, 0.6, 0.7), deliberately pick a non-greedy action (anything other than 0.7) to gather information. That is exploration.
Slide / transcript note — "each player maintains their own value table." Prof. stressed this is specific to multi-agent / game settings (covered later in the course): "every player will have their own record book." For a single-agent grid world, the one learning agent keeps one table.

How the learning update works — Temporal Difference (intuition)

Prof. drew a direct parallel to supervised learning. In ML you compute an error (target − prediction), scale it by a learning rate, and add it to the current weight. RL does the same thing across time steps — hence Temporal Difference (TD): you learn from the difference between successive time steps S1 → S2.

The universal learning one-liner (Prof.'s words) New estimate = Old estimate + α × error
TD update for a state value (research-enriched, full form in Session 3) V(s) ← V(s) + α [ R + γV(s′) − V(s) ]
↑ the bracket is the TD error (a prediction error)

The learning rate α — three cases (from the lecture)

α valueEffectProf.'s framing
α = 0No update ever; weights frozen."Will not converge, you will not learn anything at all."
α = constLearn a fixed fraction of each error.Standard hyperparameter (e.g. tuned, like in ML).
α ↓ slowlyAdaptively reduced, never 0."Tuning adaptively based on the performance" — adaptive learning.

Why this matters LAB

The same explore/exploit knob runs through all of RL: ε-greedy is still the default exploration in DQN (2015) and remains in production systems today. The TD update you just saw is the literal core of the DQN loss (the network is trained to minimise the squared TD error). The learning rate α becomes the optimiser learning rate (Adam / RMSprop) in deep RL. Recommenders (YouTube, Netflix), ad auctions (Google), and A/B testing platforms all explicitly tune an explore/exploit balance.

Research Enhanced

Why blind greedy fails — regret. Formally, cumulative regret RT = T·μ* − Σt μAt measures total reward lost vs always playing the best arm. A purely greedy agent can suffer linear regret (Θ(T)) — if its first pull of the truly-best arm is unlucky, it abandons it forever and bleeds reward every step. Well-designed explorers (ε-greedy with decay, UCB) achieve sublinear, O(log T) regret. — Auer, Cesa-Bianchi & Fischer, "Finite-time Analysis of the Multiarmed Bandit," Machine Learning 2002; Lattimore & Szepesvári, Bandit Algorithms 2020.

Worked examples

Q. From state D the action values are 0.2, 0.4, 0.6, 0.7. Which action is greedy, and which would count as exploration?

Greedy = the action leading to 0.7 (the max). Exploration = picking any other action (0.2, 0.4, or 0.6).
Key insight: "greedy" always means argmax of current estimates; "explore" = anything that is not that argmax.

Q. Apply one TD update. V(s)=0.5, V(s′)=0.8, R=0.2, α=0.1, γ=1. Find the new V(s).

  1. TD error = R + γV(s′) − V(s) = 0.2 + 1·0.8 − 0.5 = 0.5.
  2. V(s) ← 0.5 + 0.1 × 0.5 = 0.5 + 0.05.
New V(s) = 0.55. (TD error 0.5, scaled by α=0.1.)
Key insight: the bracket is exactly an ML prediction error; α controls how much of it you absorb this step.

Q. Explain, using the hotel story, why a purely greedy agent can incur linear regret, and how exploration fixes it.

  1. Suppose hotel 2 is truly best but had an off-day on your single visit → its estimate is unfairly low.
  2. Greedy locks onto hotel 1 and never re-samples hotel 2, so every future day loses (μ* − μ₁) reward.
  3. Over T days that lost gap accumulates as Θ(T) → linear regret.
  4. Exploration re-samples hotel 2 occasionally; its estimate corrects, the agent switches, and per-step loss → 0, giving sublinear regret.
Greedy's refusal to re-sample turns one unlucky observation into permanent loss (linear regret). Exploration guarantees every action is tried infinitely often, so estimates converge to truth and the agent eventually plays optimally.
Key insight: exploration is insurance against early bad luck — its small short-term cost prevents catastrophic long-term lock-in.
Exam Alert · closed book

Likely question: define exploration vs exploitation; identify greedy vs non-greedy from a value table; one-step TD update calculation.

Common mistakes: (1) thinking exploration excludes the greedy action — it doesn't (next section). (2) Forgetting that α=0 means no learning. (3) Saying TD needs a full episode — TD updates every step (that's the point).

Exam-critical: the "New = Old + α·error" one-liner and the three α cases. Prof. literally quizzed the class on α=0.

K-Armed Bandit & Action-Value Estimates

Intuition first

Now Prof. strips RL down to one state. "Let's just assume the environment is a single-state environment — there is no state change happening." You sit in front of K levers (a casino row of slot machines). Pull one → get a reward → you're in the same situation again. No future to plan, no sequence. The only question: which of the K actions has the highest payoff, and how do I find it?

The name: a slot machine is a "one-armed bandit" (one lever, it robs you). Put K of them together → k-armed bandit. A casino with 15 machines = a 15-armed bandit; with 9 machines = 9-armed. Sutton & Barto use this image for the abstract problem "pick the best of K actions."

"I'm narrowing down the problem… there's no state at all because it's only one state. All the problem I have is N actions, and from those N actions I have to choose one which will give me maximum reward." — Prof. Sangeetha — Transcript (≈ 57:30 – 58:30)

The problem, precisely

  • Repeatedly choose one of K actions.
  • Each choice returns a numeric reward from a stationary probability distribution that depends on the action.
  • Objective: maximise expected total reward over some time period (e.g. "you get 10 pulls — take home the most money").
  • One lever at a time (T1 pull lever 1, T2 lever 2, …), not simultaneously.
Stationary ≠ constant (Prof. drilled this). Stationary means the reward distribution stays fixed — e.g. a machine always pays somewhere in the range 10–100. The actual number each pull still varies. If the casino owner changed the range weekly (10–100, then 200–300, then 300–500) it would be non-stationary — and you could never pin down an arm's true value. "My reward is not constant. I'm telling the reward distribution is stationary. There's a difference."

Action-value estimate Q(a) = the sample average

The true value of an action, q*(a), is the expected reward for pulling it — but it's hidden. We estimate it by averaging the rewards we've actually seen from that action:

Sample-average action-value estimate Qt(a) = (sum of rewards received when a was taken) / (number of times a was taken)
Qt(a) = [ Σi Ri · 𝟙(Ai=a) ] / [ Σi 𝟙(Ai=a) ]
True (unknown) value we hope to converge to q*(a) = 𝔼[ Rt | At=a ]

Prof.'s live numerical example (4 slot machines, the reward each time step):

ActionRewards observedTimes takenQ(a) = mean
A1212 / 1 = 2
A23, 326 / 2 = 3
A3111 / 1 = 1
A4111 / 1 = 1

For the 6th play, A2 has the highest estimate (3) → a greedy agent pulls A2. (In the lecture Prof. first mis-stated A2 as 2.5 assuming a single 3, then corrected it to 3 once both rewards of 3 were counted — divide only by the times that action was taken.)

"We are doing this averaging process in the hope that at some time we might land on the true value of the action… we have certain laws in mathematics which tell you keep on doing a particular action and at one point you might land on the actual value." — Prof. Sangeetha — Transcript (≈ 1:23:00) [the Law of Large Numbers]

Why average a noisy signal? (a student's challenge)

A student objected: averaging makes A2 look like 2.5/3 instead of its peak 3 — isn't the average a noisy estimate that could mislead you? Prof.'s answer: in the real world reward is noisy, the future is unknown, and you don't know how much past to trust — and under all that uncertainty, the mean is the proven best estimate. You must also try each action at least once, or its Q stays 0 forever and you'll never know it.

Why this matters LAB

The bandit is the hydrogen atom of RL: simplest setting that still contains the explore/exploit tension. It's not a toy — it's deployed daily: online ad selection (each ad = an arm, click = reward), news/article recommendation (contextual bandits power many feeds), clinical trial arm allocation, and A/B testing (a bandit reallocates traffic to winners mid-test instead of splitting evenly to the end). Your webinar implements exactly this in Python.

Research Enhanced

Sublinear regret is achievable, and bandits power real systems. The required reading — Bouneffouf & Rish, "A Survey on Practical Applications of Multi-Armed and Contextual Bandits" (arXiv:1904.10040, 2019) — documents deployments across recommendation, healthcare, finance, and dialogue. Theoretically, the best arm can be identified with regret growing only as O(log T), far better than the linear regret of naive greedy. — Bouneffouf & Rish 2019; Lai & Robbins 1985 (the log-T lower bound).

Worked examples

Q. Action A2 was pulled twice, both times giving reward 3. What is Q(A2)?

Q(A2) = (3 + 3) / 2 = 3. (Sum of A2's rewards ÷ number of times A2 was taken — only A2's data.)
Key insight: never divide by total plays — divide by the count for that specific action.

Q. Reward history — A1: [2]; A2: [3,3]; A3: [1]; A4: [1]. Compute all Q estimates and state the greedy action for step 6.

  1. Q(A1)=2/1=2; Q(A2)=6/2=3; Q(A3)=1/1=1; Q(A4)=1/1=1.
  2. argmax = A2 (estimate 3).
Q = {A1:2, A2:3, A3:1, A4:1}; greedy action = A2.
Key insight: greedy reads straight off the estimate table — no recomputation of distributions needed.

Q. A casino owner resets each machine's reward range every week (10–100, then 200–300…). Explain why plain sample-averaging breaks, and what property is being violated.

  1. Sample-average weights all past rewards equally, including stale ones from old ranges.
  2. The estimate is dragged toward an outdated mean and lags reality.
  3. The violated assumption is stationarity: q*(a) is no longer fixed, so "converge to the true value" is meaningless — there's a moving target.
  4. Fix (Section 5 deep-dive): use a constant step-size α so recent rewards dominate.
Equal-weight averaging assumes a fixed distribution; under non-stationarity it converges to nothing useful. You must down-weight old data (constant-α / recency weighting).
Key insight: "stationary ≠ constant" is the exact hinge here — the range moving is what kills the average.
Exam Alert · closed book

Likely question: compute Q(a) from a reward history (e.g. A:[1,2,3,4] → Q=2.5); state why each action must be tried ≥ once; define stationary vs non-stationary.

Common mistakes: dividing by total time steps instead of per-action count; saying stationary means constant reward; forgetting an untried action stays at Q=0.

Exam-critical: the Q(a) formula, the "single state" framing, and the stationary-distribution caveat (Prof. spent ~15 minutes on it).

Greedy Action Selection

Intuition first

Greedy is the simplest possible policy: always pull the lever with the highest current estimate. No second-guessing. It's Arjun the new employee who, after trying all 10 office food stalls once, eats biryani every single day because biryani scored best on day one.

Greedy action selection At = argmaxa Qt(a)
Warm-up: try every action at least once (e.g. 4 actions → first 4 steps pull each once) so no estimate is stuck at 0.
From then on: compute Q(a) for all actions, pick the argmax, pull it. Repeat.
The flaw: one unlucky early sample of a good action permanently buries it. Arjun's noodles scored low only because the regular chef was on leave that day — greedy will never give noodles another chance.
"On day two the chef in the noodle stall was on leave… so you gave a low rating for noodles. But that doesn't mean the noodle stall is not as good as biryani. If you go with a greedy action choice, you might give away certain good actions because of always going behind the greedy." — Prof. Sangeetha — Transcript (≈ 1:30:45 – 1:31:20)
"It's a never-ending loop." A student noted greedy can loop forever on a sub-optimal arm. Prof. agreed — and that's the cue for ε-greedy (next) and UCB (next session): "How to come out of this loop is by going for ε-greedy action selection, or… UCB."

Why this matters

Greedy isn't useless — it's what you do at deployment, once learning is finished (set ε=0 to "exploit only"). The danger is being greedy while still learning. In the famous 10-armed testbed, pure-greedy plateaus well below ε-greedy because it commits too early. Greedy + one trick — optimistic initial values (start every Q high so reality "disappoints" the agent into trying everything) — recovers some exploration for free on stationary problems.

Research Enhanced

Greedy's failure is provable. On Sutton & Barto's 10-armed testbed (2000 random 10-arm problems, rewards ~ N(q*(a),1)), pure greedy (ε=0) gets stuck around ~1.0 average reward / ~35% optimal-action rate, while ε=0.1 climbs past it — empirical proof that some exploration beats none. The reason is the linear-regret lock-in from Section 2. — Sutton & Barto Ch.2, Fig 2.2.

Worked examples

Q. Estimates are Q = {A:1, B:−0.2, C:−0.5, D:−2}. Which action does pure greedy select?

Greedy selects A (the argmax, estimate 1). The negatives are ignored.
Key insight: greedy only ever looks at the single largest estimate.

Q. Arjun's averages after a month: biryani 4.4, dosa 4.1, noodles 3.0 (dragged down by one off-day). On day 31 he picks biryani. Is that greedy? What good option might he be missing, and why?

  1. Picking the max-average stall = greedy / exploitation. Yes.
  2. Noodles' true quality may be high; its low average came from a one-off bad sample (sous-chef day).
  3. Greedy never re-samples noodles, so the bad estimate is never corrected.
Yes, day-31 biryani is a greedy choice. He may be permanently missing a genuinely good noodle stall whose average was unfairly depressed by a single unlucky visit.
Key insight: greedy trusts the estimate completely — it has no mechanism to suspect an estimate is unreliable (UCB later fixes this by tracking uncertainty).

Q. Argue why pure greedy can incur linear regret even when its initial estimates are unbiased, and name two fixes from the course.

  1. Estimates are unbiased in expectation, but any single run sees finite-sample noise.
  2. If noise ranks a sub-optimal arm first early on, greedy commits to it and stops sampling the true best.
  3. Without re-sampling, the per-step gap (μ* − μ_chosen) never closes → Σ over T steps = Θ(T) = linear regret.
  4. Fixes: (a) ε-greedy — forces occasional random re-sampling; (b) optimistic initial values — inflate Q₁ so every arm gets pulled early; (also UCB, next session).
Unbiasedness doesn't protect a single trajectory: one bad early draw + no re-sampling = permanent loss = linear regret. ε-greedy and optimistic initialisation both restore the re-sampling greedy lacks.
Key insight: the problem isn't bias, it's commitment without re-checking. Exploration = mandated re-checking.
Exam Alert · closed book

Likely question: write the greedy rule (argmax Q); identify the greedy action from a table; explain greedy's weakness with the noodle/biryani story.

Common mistakes: forgetting the mandatory "try each once" warm-up; thinking greedy is always bad (it's correct after convergence / at deployment).

Exam-critical: At = argmaxa Qt(a) and the lock-in failure mode.

ε-Greedy Action Selection

Intuition first

ε-greedy is the simplest cure for greedy's lock-in: behave greedily most of the time, but with a small probability ε, throw the rulebook away and pick a random action. ε is a dial you set — ε = 0.2 means "explore 20% of the time, exploit 80%." It's Arjun thinking "biryani is my favourite, but let me give noodles another shot today, just in case that bad day was a fluke."

ε-greedy rule At = argmaxa Qt(a) with prob. (1 − ε)   [exploit]
At = random action (uniform over all K) with prob. ε  [explore] 
# Prof.'s pseudocode (Slide 17)
epsilon = 0.05   # small value to control exploration
def get_action():
    if random.random() > epsilon:
        return argmax_a(Q)        # exploit (greedy)
    else:
        return random.choice(A)    # explore (ANY action, incl. greedy)
The crucial subtlety (a student caught it). During the explore branch you choose uniformly among all K actions — the greedy one is still in the pool. So the greedy action is NOT removed during exploration. This is exactly what makes the Ex-1 probability come out to 0.75, not 0.5.

The headline derivation — Ex-1 (P that greedy is selected)

Two actions, ε = 0.5. What is P(greedy action selected)? Prof. solved this with the law of total probability:

P(greedy) via total probability P(greedy) = P(greedy | exploit)·P(exploit) + P(greedy | explore)·P(explore)
= (1)·(1−ε) + (1/K)·(ε)
= (1)(0.5) + (0.5)(0.5) = 0.5 + 0.25 = 0.75

Exploit (prob 0.5) always picks greedy → contributes 0.5. Explore (prob 0.5) picks each of the 2 actions with prob ½, so it lands on greedy an extra 0.5×0.5 = 0.25. Total = 0.75.

General formulas (K actions) P(greedy) = (1 − ε) + ε/K
P(a specific non-greedy action) = ε/K
"When I tell exploration, there is still my greedy action in the lot. You are giving the same chance to the greedy action that you give to a non-greedy action. So it is 0.25, which makes it 0.75 — the probability of choosing the greedy action." — Prof. Sangeetha — Transcript (≈ 1:57:00 – 1:57:30)

Why this matters LAB

ε-greedy is still the workhorse explorer in deep RL — the original DQN (Mnih et al., 2015) used ε-greedy with ε annealed from 1.0 → 0.1 over training. The pattern "explore lots early, exploit more later" is the decaying-ε schedule. Your webinar's Python bandit notebook implements exactly this loop: warm-up over all arms, then ε-greedy selection with incremental Q updates. Prof.: "clearly your epsilon is a hyperparameter, like your learning rate."

Research Enhanced

Fixed ε is rarely optimal — decay it. A constant ε keeps wasting ε-fraction of pulls on random actions forever, capping performance. Schedules like εt = min(1, c·K/(d²t)) (Auer et al. 2002) or simple εt=1/t achieve logarithmic regret, whereas fixed ε gives linear regret in the exploration term. Modern alternatives that explore smartly (by uncertainty, not coin-flip) — UCB and Thompson sampling — often dominate ε-greedy; both are on the course roadmap. — Auer, Cesa-Bianchi & Fischer 2002; Chapelle & Li, "An Empirical Evaluation of Thompson Sampling," NeurIPS 2011.

Worked examples

Q. ε = 0.2, K = 4 actions. What is the probability the greedy action is selected?

  1. P(greedy) = (1−ε) + ε/K = 0.8 + 0.2/4 = 0.8 + 0.05.
P(greedy) = 0.85. (And each specific non-greedy action: ε/K = 0.05.)
Key insight: always add the ε/K "bonus" — exploration can re-pick greedy.

Q. ε = 0.1, K = 3. Find P(greedy) and P(each non-greedy). Verify they sum to 1.

  1. P(greedy) = (1−0.1) + 0.1/3 = 0.9 + 0.0333 = 0.9333.
  2. P(each non-greedy) = 0.1/3 = 0.0333, and there are 2 of them.
  3. Check: 0.9333 + 2(0.0333) = 0.9333 + 0.0667 = 1.000 ✓
P(greedy) ≈ 0.933; each non-greedy ≈ 0.033; total = 1. ✓
Key insight: probabilities must sum to 1 — a fast way to catch arithmetic slips in the exam.

Q. Prove P(greedy) = (1−ε) + ε/K for K actions, then evaluate at K=2, ε=0.5. Also: what does ε=0 and ε=1 reduce the algorithm to?

  5. K=2, ε=0.5: (0.5) + (0.5)(0.5) = 0.75. ✓
  6. ε=0 ⇒ pure greedy (never explores). ε=1 ⇒ pure random (uniform, never exploits).
P(greedy) = (1−ε) + ε/K; at K=2, ε=0.5 → 0.75. ε=0 → fully greedy; ε=1 → fully random uniform selection.
Key insight: ε interpolates between two extremes — greedy (ε=0) and random (ε=1). The art is choosing/decaying ε between them.

Upper-Confidence-Bound (UCB) Action Selection

Intuition first

ε-greedy has a blind spot: when it decides to explore, it explores indiscriminately — it flips a coin and picks any non-greedy action with equal probability, with no preference for the ones that are nearly-best or the ones it's genuinely unsure about. That's wasteful. UCB fixes this by exploring with judgement: it favours actions according to their potential to actually be optimal, weighing two things at once — how close the action's estimate is to the maximum, and how uncertain that estimate still is.

Think of it as optimism in the face of uncertainty. Arjun doesn't just re-roll a random stall; he reasons "I've only tried the noodle place once — my opinion of it is shaky, so it might secretly be great." UCB gives every action an optimistic bonus proportional to how little it's been sampled. An action you've barely touched looks tempting because you're uncertain; an action you've pulled a hundred times has a tiny bonus, because you already know it well.

UCB action selection At = argmaxa [ Qt(a) + c · √( ln t / Nt(a) ) ]

Term-by-term breakdown

TermNameRole
Qt(a)Action value at tThe current sample-average estimate — the "exploit" part (closeness to the max estimate).
cConfidence levelA tunable knob ( c > 0 ) controlling how much exploration the bonus drives. Larger c → more exploration.
√(ln t / Nt(a))Measure of uncertaintyThe "explore" bonus. Grows when an action is rarely picked; shrinks the more it is sampled.

The magic is in how the two counters t (total steps so far) and Nt(a) (times this action was chosen) move:

  • Each time action a is selected → Nt(a) goes up → it sits in the denominator, so the uncertainty term shrinks. You become more confident about a, so its optimistic bonus fades.
  • Each time some other action is selected → t goes up but Nt(a) stays the same. Since t sits in the numerator (inside ln), a's uncertainty term slowly grows — the world has moved on and a is now "stale," so it deserves another look.
  • Net effect: lower-value or already-frequently-picked actions get chosen with decreasing frequency over time, but no action is ever permanently abandoned — the ln t creep eventually revives any neglected arm.

Note the asymmetry: Nt(a) pulls the bonus down fast (linear in the denominator), while ln t pushes it up slowly (logarithmic). Exploration therefore tapers off — but never quite to zero.

Why this matters

UCB is the canonical formalisation of "optimism in the face of uncertainty," one of the most reusable ideas in all of RL and bandits. It is the natural successor to ε-greedy from §05: same explore/exploit goal, but smart-exploration (steered by uncertainty) instead of a blind coin-flip. That single upgrade is why UCB powers real systems — online ad and content selection, A/B-test traffic allocation, Monte-Carlo Tree Search (the UCT rule in AlphaGo's tree is UCB applied to game-tree nodes). Where ε-greedy wastes its exploration budget uniformly, UCB spends it exactly where the agent's knowledge is thinnest.

Research Enhanced

UCB has provable, near-optimal regret. Auer, Cesa-Bianchi & Fischer's "Finite-time Analysis of the Multiarmed Bandit Problem" (UCB1, Machine Learning 2002) proved that this exact bonus formula achieves O(log T) cumulative regret — uniformly over time, not just asymptotically. That matches the famous Lai & Robbins (1985) lower bound, which showed log-T regret is the best any algorithm can do on the stochastic bandit. So UCB is not a heuristic — it is essentially optimal for this problem class. The trade-off (per Sutton & Barto): UCB is harder than ε-greedy to extend beyond bandits to general RL with large state spaces and function approximation. — Auer, Cesa-Bianchi & Fischer 2002; Lai & Robbins 1985; Sutton & Barto Ch.2.6.

Slide-35 result. On Sutton & Barto's 10-armed testbed, UCB with c = 2 performs slightly better and faster than ε-greedy with ε = 0.1 — a higher average-reward curve, especially after the early steps. The caveat from the same slide: UCB's reliance on per-action counts Nt(a) makes it awkward to scale to full RL, where you rarely revisit the exact same state.

Worked examples

Q. An action has been tried Nt(a) = 4 times, the current step is t = 50, and c = 2. Compute its UCB exploration bonus c·√(ln t / Nt(a)).

  1. ln t = ln 50 ≈ 3.912.
  2. ln t / N = 3.912 / 4 ≈ 0.978 → √0.978 ≈ 0.989.
  3. Bonus = c · 0.989 = 2 × 0.989 ≈ 1.978.
Exploration bonus ≈ 1.98. (The action's UCB score is its Qt(a) plus this ≈1.98.)
Key insight: the bonus is added on top of Q — even a mediocre estimate can win the argmax if its uncertainty bonus is large enough.

Q. At t = 100, c = 2. Arm X: Q = 1.0, N = 10. Arm Y: Q = 0.8, N = 1. Which arm does UCB select? (ln 100 ≈ 4.605.)

  1. Arm X bonus = 2·√(4.605/10) = 2·√0.4605 = 2·0.679 = 1.357 → UCB(X) = 1.0 + 1.357 = 2.357.
  2. Arm Y bonus = 2·√(4.605/1) = 2·√4.605 = 2·2.146 = 4.292 → UCB(Y) = 0.8 + 4.292 = 5.092.
  3. argmax → Arm Y (5.092 > 2.357).
UCB selects Arm Y, even though its estimate (0.8) is lower than X's (1.0). Y has been tried only once, so its huge uncertainty bonus dominates.
Key insight: UCB will repeatedly favour rarely-tried arms until their N grows enough to shrink the bonus — that's how it guarantees every arm gets sampled.

Q. (a) Why does a never-tried action ( Nt(a) = 0 ) get effectively infinite priority? (b) Why does the t in the numerator eventually "revive" a stale, long-ignored arm?

  1. (a) With Nt(a) = 0 the term ln t / Nt(a) divides by zero → the bonus → +∞. By convention UCB treats an unsampled action as having maximal value, so it is selected first. This forces the mandatory "try every arm at least once" warm-up automatically — no special-case code needed.
  2. (b) Suppose arm a is ignored for a long stretch. Its Nt(a) is frozen, but every step elsewhere increments t. Inside the bonus, ln t keeps creeping up while the denominator is fixed, so √(ln t / Nt(a)) slowly grows. Eventually the bonus is large enough that a wins the argmax and gets re-sampled.
  3. Together: (a) guarantees full initial coverage; (b) guarantees no arm is ever permanently abandoned — both are why UCB explores every action infinitely often and its estimates converge to q*(a).
(a) N = 0 ⇒ division by zero ⇒ infinite bonus ⇒ untried arms are picked first (automatic warm-up). (b) ln t in the numerator grows while N stays fixed, so a neglected arm's bonus inflates over time until it is re-tried — preventing permanent lock-out and ensuring convergence.
Key insight: the two counters encode opposite pressures — Nt(a) builds confidence (shrinks the bonus), t restores doubt (grows it). Their balance is exactly "optimism in the face of uncertainty."

Optimistic Initial Values

Intuition first

Every method so far is biased by where you start it — the number you wrote in Q1(a) before pulling anything leaks into your estimates. Optimistic initial values turn this "bug" into a free exploration trick: instead of starting every action at a neutral 0, start them all wildly high (in the 10-armed testbed, Q1=+5 when true values hover near 0).

Picture a greedy agent given a head-start rumour that every stall in the food court is a 5-star Michelin kitchen. It is pure greedy — no ε, no coin flips — yet it still ends up tasting everything. Why? Reality can only disappoint it. Whatever it tries first pays less than the inflated 5 it expected, so that action's estimate drops below the untried ones, and greedy is yanked toward the next shiny untouched lever. The optimism itself manufactures exploration with zero extra parameters.

Constant-α update with an optimistic start Qn+1 = Qn + α [ Rn − Qn ] ,  set Q1 = +5 (≫ q*(a) ≈ 0)
unrolled: Qn+1 = (1−α)n Q1 + Σi α(1−α)n−i Ri

The very first term (1−α)n Q1 is the optimism carried forward — it is the bias. With constant α it never fully reaches 0 (it only shrinks geometrically), so under constant-α the initial-value bias is permanent but decreasing. Under a sample-average (1/n) step the same term vanishes the moment every action has been pulled at least once.

The "disappointment" mechanism, step by step

Inflate every estimate. Q1(a)=+5 for all K arms — all equally, absurdly attractive.
Greedy pulls arm a. Tie-broken arbitrarily; it expected +5.
Reality underdelivers. Real reward (≈0) < the inflated 5, so the update pushes Q(a) down toward reality — arm a is now worth less than the still-untouched 5s.
Greedy switches. argmax now points at a fresh untried arm. Repeat → all actions get tried several times before any estimate settles.

Why this matters

This buys you systematic early exploration for free — no ε, no random coin, zero extra hyperparameters — on stationary tasks. It is the cheapest possible nudge that breaks greedy's lock-in (§s4). Contrast it with UCB (§s6), which also explores without random flips but does so by tracking each arm's uncertainty on every step: UCB keeps re-checking under-sampled arms forever, whereas optimism is a one-time opening push that is spent once the inflated estimates have decayed. That single difference is exactly why UCB survives in non-stationary settings and optimism does not.

Research Enhanced

Sutton & Barto Ch.2, Fig 2.3. Optimistic-greedy (Q1=5, ε=0) is plotted against realistic ε-greedy (Q1=0, ε=0.1) on the 10-armed testbed. The deeper reason the two biases behave differently is the Robbins–Monro conditions for stochastic approximation: a step-size sequence converges iff Σn αn = ∞ and Σn αn2 < ∞. The sample-average step αn=1/n satisfies both (Σ1/n diverges, Σ1/n² converges) → estimates converge and the initial bias is wiped out. A constant α satisfies the first but violates the second (Σα² = ∞) → it never fully forgets, so the optimism bias persists (while shrinking). — Sutton & Barto, RL: An Introduction 2nd ed., Ch.2; Robbins & Monro 1951.

Caution — it is only a trick. Prof.'s slide is blunt: optimistic initial values are a simple trick, effective on stationary problems, but far from a generally useful exploration method. All the exploration happens in a burst at the very beginning; once the inflated estimates have decayed to reality there is no built-in mechanism left to keep exploring. Don't reach for it as your default explorer — reach for it only when the problem is stationary and you want a one-line, parameter-free head start.

Worked examples

Q. Optimistic start Q1=5, constant α=0.5. The first pull of this arm returns reward R1=1. Compute Q2, and explain what it triggers.

  1. Q2 = Q1 + α[R1 − Q1] = 5 + 0.5·(1 − 5).
  2. = 5 + 0.5·(−4) = 5 − 2 = 3.
Q2 = 3. The estimate dropped from 5 toward the real reward — this arm (now 3) is worth less than every untried arm still sitting at 5, so the greedy argmax abandons it and pulls a fresh arm. That is the disappointment-driven switch.
Key insight: because reality is below the inflated start, the update is negative — optimism guarantees the first-pulled arm loses its lead, forcing a sweep through all arms.

Q. Why does a sample-average estimate make the initial-value bias vanish, while a constant-α estimate keeps it forever (only fading)?

  1. In both, the leftover initial term is the weight on Q1. Unroll the update: sample-average weights Q1 by ∏(1−1/k), constant-α weights it by (1−α)n.
  2. Sample-average: on the very first pull of an action the step is α1=1/1=1, so (1−α1)=0 — the Q1 term is multiplied by 0 and annihilated the instant the action is selected once.
  3. Constant-α: (1−α)n with fixed α∈(0,1) is never exactly 0 for any finite n — it shrinks geometrically but a sliver of the optimism always remains.
Sample-average's first step has α=1, which zeroes out Q1 as soon as the action is tried once → bias gone. Constant-α multiplies Q1 by (1−α)n, which decays toward but never reaches 0 → bias is permanent yet ever-decreasing.
Key insight: this is the Robbins–Monro story in miniature — 1/n shrinks the step (Σα²<∞) so memory of the start dies; constant α refuses to shrink, so it never fully forgets.

Q. (Slide's open question.) Explain rigorously why optimistic initial values fail to explore adequately in a non-stationary problem.

  1. Optimism is a one-shot impulse. The exploration comes entirely from the gap between Q1=+5 and reality. That gap is consumed during the opening phase, after which the inflated term (1−α)nQ1 has decayed away and every estimate sits near its true value.
  2. Non-stationarity means q*(a) drifts later. Suppose at step 5000 a previously-mediocre arm's true value rises above the current best. There is no fresh optimism left to flag it — the estimates have long since settled.
  3. Pure greedy provides no recovery. With ε=0 the agent only ever pulls its current argmax, so the newly-improved arm is never re-sampled, its stale low estimate is never corrected, and the drift goes undetected — the agent locks onto a now-suboptimal arm (linear-regret lock-in returns).
  4. Contrast: mechanisms that explore continuously — decaying-but-nonzero ε, or UCB's ever-growing √(ln t / Nt(a)) uncertainty bonus for un-pulled arms — keep probing forever and so catch the drift. Optimism cannot, because its push is spent at t≈0.
Optimistic init front-loads all its exploration into the start. Once the inflated estimates decay to reality, a greedy agent has no remaining drive to explore; when q*(a) later changes (non-stationary), nothing triggers re-sampling, the change is missed, and the agent stays stuck on a no-longer-optimal action. Adequate exploration in non-stationary settings needs an ongoing source (constant-α recency weighting plus continual exploration via ε or UCB), not a one-time optimistic kick.
Key insight: the failure is temporal — exploration that is concentrated at the start cannot react to information that arrives later. Non-stationarity demands perpetual, not opening, exploration.

Policy-Based Methods: Softmax & Gradient Bandit

Intuition first — preferences, not values

Every method so far (greedy, ε-greedy, UCB) shares one habit: estimate Q(a), then act on it. Policy-based methods make a clean break — forget the value estimates entirely. We stop asking "how much is each action worth?" and ask only the question we actually care about: which action should I choose?

The trick is to attach a numerical preference Ht(a) to each action. A preference is not a reward or a value — it has no units, it isn't an estimate of anything. It is just a learnable "how much do I lean toward this action" knob. Higher H ⇒ pick it more often. Learning = nudging the preferences so the leaning matches what pays off.

Why this is a big deal. Value-based RL learns numbers about actions and derives a policy as an afterthought (argmax). Policy-based RL learns the policy directly. This is the conceptual seed of policy-gradient / actor-critic methods (REINFORCE, A2C, PPO) — the entire right-hand branch of the deep-RL family tree starts right here, in the bandit.

From preferences to a policy: hardmax vs softmax

Given preferences, how do we turn them into an action? The naïve route is hardmax — just take the argmax:

Hardmax selection (the naïve route) At = argmaxa Ht(a)

Hardmax has two fatal flaws. (1) It is deterministic — same preferences ⇒ same action every time ⇒ no exploration. (2) It is not differentiable: nudging H(a) a little does nothing to the output until it crosses the max and the choice snaps, so ∂Pr(a)/∂H(a) = NA. With no gradient there is nothing to do gradient ascent on. We need a smooth bridge from preferences to probabilities.

Softmax — preferences → probability distribution

Softmax exponentiates each preference and normalises, producing a valid probability distribution over the actions:

Softmax (Boltzmann) action probabilities Pr{At = a} = eHt(a) / ( Σa′ eHt(a′) ) = πt(a)

This single move buys us both things hardmax lacked: every action gets a non-zero probability (exploration ✓), and the function is smooth, so the gradient ∂Pr(a)/∂H(a) exists (differentiable ✓). A softmax over preferences is a differentiable policy — the foundational object of all policy-gradient methods.

Prof.'s numeric example — three actions with preferences H = (6, 9, 2):

softmax of the preference vector (6, 9, 2) Ht = ( 6, 9, 2 )
→ softmax = ( 0.047, 0.952, 0.00087 )
# Pr ≈ 0.95 → a2, Pr ≈ 0.05 → a1, Pr < 0.01 → a3

Notice softmax is not winner-take-all: a2 dominates (preference 9) but a1 still gets a real ~5% share, and even the worst action a3 keeps a tiny <0.1% chance. That residual probability is the exploration — and because the gap is exponential, a 3-point lead (9 vs 6) already translates into a 20× probability ratio.

# Differentiability: hardmax vs softmax
# Suppose a1 starts giving good relative performance.
# HARDMAX  A_t = argmax_a H(a)         d Pr(a1) / d H(a1) = NA   (flat, then a jump)
# SOFTMAX  Pr(a1) = e^H(a1) / Σ e^H(a')  d Pr(a1) / d H(a1) EXISTS  -> we can ascend it

Gradient bandit — learning the preferences

Because the policy is differentiable, we can do gradient ascent on expected reward: after taking At and seeing Rt, nudge the preferences in the direction that makes good actions more probable. The update compares the reward to a baseline R̄t (the running mean of all rewards so far):

Gradient bandit update — α = step size, πt(a) = softmax prob., R̄t = baseline Ht+1(At) = Ht(At) + α (Rt − R̄t) (1 − πt(At)) # the chosen action
Ht+1(a)    = Ht(a)    − α (Rt − R̄t) πt(a)       # ∀ a ≠ At

Read it as a tug-of-war driven by the sign of (Rt − R̄t):

  • Reward beats the baseline (Rt > R̄t): push the chosen action's preference up, and pull every other action's preference down. "That was better than my average — do it more."
  • Reward below the baseline (Rt < R̄t): do the opposite — push the chosen action down, others up. "That was worse than my average — back off it."

Why the baseline matters — the results

The baseline R̄t is not optional decoration. It re-centres rewards so the update reacts to relative performance, not raw magnitude — which sharply reduces the variance of the gradient estimate. On the 10-armed testbed the difference is dramatic:

Settingα = 0.1α = 0.4
With baseline R̄t~85% optimal~73% optimal
Without baseline~48% optimal~27% optimal

Same algorithm, same step sizes — adding the baseline roughly doubles the % optimal-action rate. The baseline substantially improves performance.

Why this matters

Softmax-over-preferences is your first differentiable policy, and that is the entire idea behind policy-gradient methods. Swap the K preference scalars for a neural network πθ(a|s) and the gradient-bandit update generalises into REINFORCE (Williams 1992) and then actor-critic — the actor in A2C/A3C, TRPO, PPO is exactly this softmax policy trained by gradient ascent. And the humble baseline R̄t is the direct ancestor of the advantage term A(s,a)=Q−V used by every modern actor-critic: subtracting a baseline to cut variance is the same trick, scaled up. The bandit is where the entire policy-based branch of deep RL is born.

Research Enhanced

Q. Two actions have preferences H = (2, 0). Compute the softmax probabilities. (Use e2≈7.389, e0=1.)

  1. Numerators: e2≈7.389, e0=1. Denominator = 7.389 + 1 = 8.389.
  2. π(a1) = 7.389 / 8.389 ≈ 0.881; π(a2) = 1 / 8.389 ≈ 0.119.

// deep dives

Conceptual Deep Dives

A · Reward vs Value — why both are necessary

A Markov Reward Process assigns each state a value equal to the expected discounted return. Values satisfy the Bellman equation — a self-consistency between a state and its successors:

V(s) = 𝔼[ Rt+1 + γ V(st+1) | st=s ]

Reward alone is the Rt+1 term; value adds the discounted future γV(s′). Drop the future term and you're back to greedy-on-reward — blind to the chess checkmate. The bandit is the degenerate case: one state, γ irrelevant, so value collapses to "expected immediate reward" = q*(a).

B · The dilemma quantified — regret

Cumulative regret over T rounds RT = T·μ* − Σt=1..T μAt
  • Pure greedy: can be Θ(T) — linear regret (one unlucky start → permanent loss).
  • ε-greedy (fixed ε): linear in the exploration term (wastes ε·T random pulls forever).
  • ε-greedy (decaying ε) / UCB: O(log T) — sublinear; provably near-optimal (Lai–Robbins lower bound is also log T).

C · Sample average vs constant step-size (the non-stationary fix)

The sample average can be computed online without storing history — derive the incremental form:

Incremental update (derivation) Qn+1 = (1/n) Σi=1..n Ri = (1/n)( Rn + (n−1)Qn ) = Qn + (1/n)[ Rn − Qn ]

The step-size here is 1/n (shrinks → true average, great for stationary). For non-stationary problems, replace it with a constant α:

Constant-α → exponential recency-weighted average Qn+1 = Qn + α[Rn − Qn] = (1−α)nQ1 + Σi α(1−α)n−i Ri

The weight on reward Ri is α(1−α)n−i — it decays exponentially the older the reward. Recent rewards dominate, so the estimate tracks a drifting q*(a). This is the concrete fix for the casino-changes-the-range scenario from Section 3. Convergence rule (Robbins–Monro): Σαn=∞ and Σαn²<∞ — satisfied by 1/n (converges), violated by constant α (keeps adapting, by design).

The bridge is associative (contextual) tasks: a bandit where the best action depends on a context/situation — "a mapping from situations to the actions that are best." Add state transitions on top and you have a Markov Decision Process, the subject of the rest of the course.

# ε-greedy survives unchanged into DQN action selection
def select_action(state, q_net, epsilon):
    if random.random() < epsilon:
        return env.action_space.sample()     # explore
    return q_net(state).argmax().item()      # exploit: argmax_a Q(s,a)

E · Beyond ε-greedy — UCB, Thompson, softmax

UCB — optimism in the face of uncertainty (next session) At = argmaxa [ Qt(a) + c·√( ln t / Nt(a) ) ]
  • UCB: adds a bonus that's large for rarely-tried actions → explores by uncertainty, not coin-flip. Beats ε-greedy on bandits; harder to scale to deep RL.
  • Thompson sampling: Bayesian — keep a posterior over each arm's value, sample from it, act greedily on the sample. Excellent empirical performance.
  • Softmax / Boltzmann: pick action a with prob ∝ eQ(a)/τ; temperature τ tunes exploration smoothly (a differentiable policy → leads into policy-gradient methods).

F · The learning rate α in deep RL

The α in TD is conceptually the same α that becomes the optimiser learning rate (Adam, RMSprop) when Q is a neural net. The DQN loss is literally the squared TD error (R + γ maxa′Q(s′,a′) − Q(s,a))²; gradient descent with learning rate α nudges the network weights down that error — the same "New = Old + α·error" Prof. wrote, now applied to millions of parameters.
