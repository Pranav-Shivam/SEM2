What Are Edges? — Low-Level Vision

The Brain Analogy: The Schrödinger Cat Experiment

Prof opened the lecture with a remarkable biological precedent that anchors the entire field of edge detection in neuroscience. In the 1950s, Schrödinger exposed a cat to radioactive material and immediately dissected its brain. Within the cat's neural tissue, he discovered neurons that fired specifically and selectively in response to oriented edges — separate neurons detecting edges at precisely 0°, 45°, 90°, and 180°.

This finding is the biological origin of edge detection as the fundamental building block of visual processing. The visual cortex does not process raw pixel intensities — it processes oriented edge responses. Every classical computer vision algorithm, and every modern deep learning model, starts from this exact insight: to understand an image, first find its edges.

This is not a coincidence or an analogy — it is a discovered equivalence. When you train a convolutional neural network from scratch on ImageNet, the first-layer kernels converge to oriented edge detectors. Backpropagation independently rediscovers what evolution spent millions of years building into the mammalian visual system.

Block 1 — Edge Detection: Every CNN's first layer kernels detect oriented edges. Prof was explicit: "We are going to have 32 kernels of size 3×3. What does those kernels do? They formulate N number of permutations of this only. They detect edges at 45 degree, another detects at 90 degree, another detects at minus 45 degree." This is Session 4's entire topic — what Block 1 learns and how it is computed classically.

Block 2 — Textures: Adjacent detected edges combine to form local texture patterns. The texture of skin, fabric, metal, wood — all are describable as specific statistical arrangements of oriented edges. A smooth surface produces uniform edge responses; a rough surface produces scattered multi-orientation responses.

Block 3 — Patterns: Textures assemble into recognisable object-specific patterns. The canonical example: two circular textures (eye-shape) at a fixed spatial distance produce a "pair-of-eyes" pattern. This is no longer a texture — it is a geometric arrangement with semantic significance.

Block 4 — Parts of Object: Patterns combine into semantic parts. "Eye pattern + nose pattern + mouth pattern + oval boundary pattern = face region." This block handles compositional structure — how parts relate spatially to form objects.

Final Layer — Complete Object: The object classification or detection head maps assembled parts to class labels. This is the only layer that "knows about" categories in the traditional sense. Every layer below is purely structural.

Prof noted that four is the minimum. For special modalities — MRI with depth information, 3D point clouds, medical volumetric data — additional blocks handle domain-specific features: depth estimation, density maps, Gaussian 3D splatting, NeRF representations. The hierarchy extends, it does not change its character.

Slide 3 — Task Definition

Slide 3 states the core task of edge detection with precision: "Convert a 2D image into a set of curves (made of edges)." This is a precise mathematical formulation — edge detection is an image-to-curve transformation, not a classification or regression task in the traditional sense. The output is a binary or graded edge map. This framing matters: it tells you that the output space is structured (curves, not arbitrary point clouds) and that continuity is a constraint the algorithm should respect.

Formal Definition: What Exactly Is an Edge?

Two complementary definitions are used throughout the session, and understanding both is important because they motivate different algorithmic approaches:

Definition 1 (Spatial): An edge is a place of rapid change in the image intensity function F(x, y). "Rapid" is defined relative to a threshold — the absolute value of the intensity gradient must exceed some chosen value.

Definition 2 (Directional): An edge is a pixel where intensity variation is significant in one specific direction. That direction can be the x-axis, y-axis, or any combination — a vector in XY space. This is the key distinction: an edge has both a location and an orientation.

The four physical causes of edges in images arise from fundamentally different scene properties, yet all produce the same mathematical signature — a large gradient magnitude:

Three Requirements for a Good Edge Detector — The Design Contract

These three requirements come directly from Slide 7 and were discussed at approximately the 35-minute mark in the transcript. They are not arbitrary — they are the mathematical specification that the Canny algorithm was explicitly designed to satisfy. Understanding the requirements is understanding why each step of Canny exists.

Requirement 1 — Good Detection: The filter must respond to edges and not to noise. This means a low probability of missing real edges (high sensitivity / low false negatives) AND a low probability of false edges generated by noise (high specificity / low false positives). These two demands are in tension — maximum sensitivity also amplifies noise. The resolution is Gaussian smoothing before differentiation.

Requirement 2 — Good Localization: The detected edge must be as close as possible to the true physical edge location in the scene. The edge should be a single pixel wide — not a thick blob, not a diffuse region. Gaussian smoothing (applied for Requirement 1) unfortunately spreads and smears edge responses, worsening localization. This tension is resolved by Non-Maxima Suppression (NMS).

Requirement 3 — Single Response: There should be only ONE detected edge point for each true physical edge point. Not multiple fragmented detections for the same edge, not a cluster of hits where one hit should be. This is enforced by double-threshold hysteresis.

Worked Examples SELECT DIFFICULTY
A pixel has intensity 200. Its right neighbour has intensity 210. Its bottom neighbour has intensity 201. Is this pixel an edge? Justify using the formal definition.

  1. Compute the intensity changes: horizontal difference = |210 − 200| = 10. Vertical difference = |201 − 200| = 1. These are the discrete approximations to ∂F/∂x and ∂F/∂y respectively.

  2. Contextualise the magnitudes. Out of the maximum possible intensity range (0–255), changes of 10 and 1 are both small. The gradient magnitude |∇F| = √(10² + 1²) = √101 ≈ 10.05.

  3. Apply threshold = 20 (typical for relatively noise-free images): |∇F| = 10.05 < 20. NOT an edge by this threshold. The intensity change is within the noise band.

  4. Apply threshold = 5 (highly sensitive setting): |∇F| = 10.05 > 5. IS a weak edge by this threshold. The dominant direction is horizontal (x-direction), since 10 >> 1. Edge orientation θ = arctan(1/10) ≈ 5.7° from horizontal.

AnswerDepends on threshold. With threshold=20: NOT an edge (|∇F|=10.05 < 20). With threshold=5: weak edge in horizontal direction. Edge detection is inherently threshold-dependent.
Key Insight

The word "abrupt" in "abrupt intensity change" is not absolute — it is relative to a threshold that you choose as a hyperparameter. This is why threshold selection is a core algorithmic problem, not a detail. Canny's double-threshold system exists precisely because no single threshold is universally correct.

This example demonstrates the most fundamental property of gradient-based edge detection: the binary "edge / not edge" decision is a hard threshold applied to a continuous quantity. The same pixel can be classified as an edge or not depending solely on your threshold choice. In practice, this motivates the use of two thresholds (Canny's hysteresis) rather than one, and motivates learned thresholds (CNN training) rather than hand-set ones.
Prof said the CNN hierarchy is: Layer 1 = edges, Layer 2 = textures, Layer 3 = patterns, Layer 4 = parts. A ResNet-50 is being used to classify handwritten digits (MNIST). Describe what each of the 4 blocks likely learns for the digit '8'.

  1. Block 1 — Oriented Edges: The two loops of '8' each consist of curved stroke segments cycling through all orientations 0°–360°. Block 1 learns to detect: horizontal strokes (tops and bottoms of loops), vertical strokes (sides of loops), diagonal strokes at ±45°, and crucially, curved-edge segments at intermediate orientations. The 32 kernels collectively tile the orientation space.

  2. Block 2 — Textures: Adjacent edge responses combine into local texture signatures. The circular/oval texture (a ring of edges closing on itself) appears twice. The figure-eight crossing point produces a unique multi-direction texture not found in other digits. The interior regions (inside the loops) produce a flat "no-edge" texture.

  3. Block 3 — Patterns: An "oval loop with tight curvature" pattern is assembled twice — once for the top loop (smaller, in '8') and once for the bottom loop (slightly larger). The crossing-point pattern connects them. This is the first level where the algorithm "knows" about loops as objects rather than just edge statistics.

  4. Block 4 — Parts: "Two adjacent closed loops sharing a single crossing point, with the top loop smaller than the bottom loop" = the semantic structural description of digit '8'. This is compared against the learned part-templates for all 10 digits. Digits '0' (one large loop), '6'/'9' (one asymmetric loop with a tail), and '3' (two open arcs) are discriminated here.

AnswerBlock 1: Curved strokes at all orientations. Block 2: Circular/oval loop textures. Block 3: "Closed oval loop" pattern (×2). Block 4: Two-loop topology with crossing point — the defining structure of '8'.
Key Insight

This explains why '8' is harder to classify than '1'. The digit '1' is resolved almost entirely by Block 1 (a single dominant vertical edge). The digit '8' requires correct assembly across all four blocks. This is also why dropout regularisation in deep networks disproportionately hurts complex-digit accuracy — it disrupts the part-assembly process in the higher blocks.

The 4-block analysis is not just conceptual. You can empirically verify it by examining the feature visualisations (maximally activating inputs) for each layer of a trained MNIST network. Tools like Lucid (Google Brain) or feature visualisation via activation maximisation confirm: Layer 1 shows oriented edges, Layer 2 shows texture patches, Layer 3 shows curve segments, Layer 4 shows loop-shaped templates. This is one of the most replicated findings in deep learning interpretability research.
Prof mentioned: "We do second-order derivatives in case of mid-level vision like corner detection." Explain WHY first-order derivatives are sufficient for edge detection but second-order is needed for corner detection, using mathematical reasoning.

  1. First-order derivative at an edge: At an ideal step edge, ∂F/∂x or ∂F/∂y has a single sharp peak at the edge location. The gradient vector ∇F = (∂F/∂x, ∂F/∂y) has large magnitude |∇F| at the edge, and near-zero magnitude everywhere else (flat regions). This gives both edge location (where |∇F| is maximum) and edge orientation θ = arctan(∂F/∂y / ∂F/∂x). The three requirements are all satisfiable with first-order derivatives alone.

  2. The problem: edges also have large first-order derivatives: A corner is a point where edges from two different orientations meet. At a corner, |∇F| is large — but so is it at every point along either edge leading into the corner. A first-order detector cannot distinguish "corner = intersection of two edges" from "edge = single directional change". Both produce large gradient magnitudes.

  3. Second-order derivatives — the Hessian matrix: The Hessian H at pixel (x,y) is H = [[∂²F/∂x², ∂²F/∂x∂y], [∂²F/∂x∂y, ∂²F/∂y²]]. The eigenvalues λ₁, λ₂ of H encode how the gradient changes in each direction. At a flat region: λ₁ ≈ 0, λ₂ ≈ 0. At an edge: λ₁ large, λ₂ ≈ 0 (change in one direction). At a corner: λ₁ large, λ₂ large (change in two directions).

  4. Harris Corner Detector (1988): Harris is defined exactly by this analysis. The Harris score R = det(H) − k·trace(H)². Since det(H) = λ₁λ₂ and trace(H) = λ₁ + λ₂: when both eigenvalues are large (corner), R >> 0. When only one is large (edge), R < 0. When both are small (flat), R ≈ 0. This three-way discrimination is impossible with first-order information.

  5. Why second-order is expensive and deferred: Computing the full Hessian requires three second-order partial derivatives, an eigendecomposition per pixel, and filtering at multiple scales. Prof explicitly acknowledged this: second-order derivatives are reserved for when corner-level precision is necessary, not used by default — because both time and space complexity increase.

AnswerFirst-order = one gradient peak per edge = sufficient for edges (location + orientation from ∇F). Second-order (Hessian eigenvalues) needed to distinguish edge (one large eigenvalue) from corner (two large eigenvalues) — impossible with ∇F alone. Directly the Harris Corner foundation.
Key Insight — Spoiler for Session 5+

When you see the Harris Corner Detector in a future session, you will recognise it as: M = Σ w(x,y) [∇F∇Fᵀ] (the structure tensor), R = det(M) − k·trace(M)², thresholded to find corners. Every symbol is a direct consequence of the reasoning above. You have already derived the intuition.

A subtlety: Prof referred to the Hessian as "second-order derivatives," correct for the formal definition. However, Harris (1988) actually uses the structure tensor (second moment matrix), computed from first-order derivatives: M = Σ w [[Ix², IxIy],[IxIy, Iy²]]. The Hessian-based corner detector is technically the Beaudet detector (1978). Both require two large eigenvalues to signal a corner — the conceptual argument applies to both.

Step Edge

Definition: Intensity abruptly jumps in a single pixel step, e.g., 0 → 255.

Profile: [0, 0, 0, 255, 255, 255]

1st derivative: [0, 0, 255, 0, 0] — single spike

Ramp Edge

Definition: Intensity changes gradually over multiple pixels.

Profile: [0, 0, 50, 100, 150, 200, 255, 255]

03 Gradient Computation — The Mathematics of Edges

Two Options for Differentiation

Option 1 — Analytical Derivative

Reconstruct the continuous function f(x,y) from discrete samples, then compute the analytical (exact) partial derivatives.

Verdict: Impractical. Reconstruction from discrete pixels is ill-posed and computationally prohibitive.

Option 2 — Discrete Finite Difference

Approximate derivatives using neighboring pixel differences directly on the discrete grid.

Verdict: This is what every real edge detector uses — Sobel, Prewitt, Canny, deep CNNs. Simple, fast, effective.

The gradient vector ∇F = [GX, GY] encodes two properties simultaneously: magnitude |G| (edge strength — how sharp the transition) and direction α (the direction of steepest intensity increase, always perpendicular to the edge itself).

Slide 9-10 — Prof. Dhruba, Axis Convention Warning

Prof explicitly warned: "Within your textbooks, x-axis is taken as a vertical axis and y-axis is a horizontal axis — opposite to what Indians are attuned to." Standard image coordinates: row index increases downward (x = vertical), column index increases rightward (y = horizontal). Transcript (~48:30). In the exam, always use x = vertical (row), y = horizontal (column).

Discrete Partial Derivatives (Finite Difference)
GX = F[x+1, y] − F[x, y]   (change along x = vertical direction = next row)
GY = F[x, y+1] − F[x, y]   (change along y = horizontal direction = next column)
|G| = √(GX² + GY²)   [exact gradient magnitude]
|G| ≈ |GX| + |GY|   [L1 approximation — faster, avoids sqrt]
α = arctan(GY / GX)   [gradient direction — use atan2 in code]
Edge direction = α + 90°   [edge runs PERPENDICULAR to gradient]
Special case: GX=0 → α = 90° [pure horizontal edge]

Why This Matters

GX and GY are literally what your GPU computes in the backward pass of every CNN. The gradient ∇F of the image is the foundation of: Sobel edge detection in OpenCV (cv2.Sobel), gradient magnitude as perceptual loss in style transfer (Gatys 2015), total variation regularisation in image reconstruction. Gradient Operators — Roberts, Prewitt & Sobel

Why Kernels? From Pixel-by-Pixel to Convolution

Computing gradients pixel-by-pixel using nested loops is O(M²N²) in both time and space — completely impractical for real images. We already know spatial filtering (convolution) from Session 3: slide a kernel over the image, multiply-and-sum. The key insight here is that we can approximate GX and GY using small convolution kernels. Instead of nested loops referencing arbitrary pixel pairs, one matrix convolution gives us the gradient at every pixel simultaneously. This is the same computational paradigm that underpins modern CNNs — the gradient filters you are learning here are the direct ancestors of learned convolutional feature detectors.

The general gradient approximation framework: define a small kernel (2×2 or 3×3), convolve it with the image, and the output at each position is the local gradient component GX or GY. The kernels differ by how they weight neighbouring pixels — and those weighting choices determine noise sensitivity, localization accuracy, and detection strength.

Roberts Cross Operator (2×2)

The Roberts Cross operator, introduced by Lawrence Roberts in 1963, is the earliest gradient operator used in computer vision. It uses a 2×2 kernel rather than the 3×3 kernels that came later, making it the fastest of the classic operators. Roberts computes diagonal differences: GX captures edges at 45°, GY captures edges at −45°.

GX kernel: [[-1, 0], [0, +1]] — differences along the main diagonal (top-left to bottom-right). GY kernel: [[0, -1], [1, 0]] — differences along the anti-diagonal.

Characteristics:

  • Excellent localization — responds immediately when an edge occurs, no averaging over large neighbourhood
  • Very sensitive to noise — tiny 2×2 kernel provides zero smoothing; a single noisy pixel triggers a false response
  • Poor detection of gradual edges — only samples 2 pixels, misses wide transitions
  • Fastest computation — 2×2 = 4 multiply-adds vs 9 for 3×3 kernel
  • Diagonal orientation — detects 45° and −45° edges most strongly, rather than horizontal/vertical

Prewitt Operator (3×3 — Uniform Weights)

The Prewitt operator, introduced by Judith M. S. Prewitt in 1970, scales up to a 3×3 kernel with uniform weights. By comparing entire rows (for GX) or entire columns (for GY), Prewitt achieves implicit averaging across the perpendicular direction — providing some noise reduction that Roberts completely lacks.

GX kernel detects horizontal edges (intensity change top-to-bottom): compares the entire bottom row against the entire top row. The middle row contributes zeros — only the extremes count. Formula: Gx = (z7+z8+z9) − (z1+z2+z3).

GY kernel detects vertical edges (intensity change left-to-right): compares the right column against the left column. Formula: Gy = (z3+z6+z9) − (z1+z4+z7).

Characteristics:

  • Some noise reduction — 3×3 averaging over three pixels per row/column provides implicit smoothing
  • Uniform weights — no center emphasis means the center row/column pixel z5 receives exactly the same weight as corner pixels
  • Better detection than Roberts — samples more pixels, more robust to single noisy pixels
  • Moderate localization — 3×3 neighbourhood blurs the edge response slightly compared to Roberts

Sobel Operator (3×3 with Center Weighting)

The Sobel operator, introduced by Irwin Sobel & Gary Feldman in 1968 (at the Stanford AI Lab), improves on Prewitt by a critical insight: weight the center row/column with factor 2. This seemingly small change does two things simultaneously: it amplifies the signal at the exact edge location (improving detection of subtle edges), and it applies a Gaussian-like smoothing in the perpendicular direction (improving noise immunity).

GX kernel: center column weight = ±2. Formula: Gx = (z7+2·z8+z9) − (z1+2·z2+z3). GY kernel: center row weight = ±2. Formula: Gy = (z3+2·z6+z9) − (z1+2·z4+z7).

Why the "2" matters (Prof's signal amplification argument): Consider a very fine edge between pixel values 10 and 20. After Gaussian smoothing, this becomes 8 and 12 — the variation is only 4, easily confused with noise. But if the center weights are 2×, the contribution from the direct centre pixel is doubled, amplifying the signal so it clearly rises above the noise floor. Sobel sacrifices some localization precision (the response is slightly wider than Roberts) to gain this noise robustness.

Characteristics:

  • Best noise immunity of the three classical operators — center weighting provides implicit Gaussian smoothing
  • Best detection sensitivity — 2× amplification lifts subtle edges above noise threshold
  • Moderate localization — slightly broader response than Roberts due to 3×3 kernel and smoothing
  • Thicker edge outputs — the smoothing effect widens the detected edge band
  • De facto standard — most widely used first-derivative edge operator in production CV systems

Canny Step 1 — Gaussian Smoothing

Why Smooth First?

Raw images have noise (pixelation, sensor jitter). If you compute gradients directly, noise creates false edges everywhere. Prof showed: a noisy constant region has tiny but non-zero differences at every pixel, so no clear extremum appears — you can't find any edge.

Solution: convolve with Gaussian kernel first. This weighted average blurs out high-frequency noise while preserving significant intensity changes (true edges).

Prof example from transcript (~1:44): "You have values like 0, 10, 20, 30, 10, 0. Twenty, 30 — constant jittering. When we take the gradient of this: 0−10=10, 10−20=10, 30−20=20. No extrema. You smoothen out the image by multiplying with a Gaussian filter. Post that, all this noise is reduced."

The Convolution Associativity Trick

This is a clever optimisation Prof highlighted. Instead of:

1. Compute (F * G) [convolve image with Gaussian] — expensive
2. Compute ∂/∂x(F * G) [take derivative of result] — second operation

Use the associativity of convolution:
∂/∂x (F * G) = F * (∂G/∂x)

This means: take the partial derivative of the KERNEL itself (cheap — it's 3×3), then convolve with the image. One operation instead of two.

Canny Step 2 — Gradient Magnitude & Direction

Computing Gradients After Smoothing

After Gaussian smoothing, compute GX and GY at every pixel using Sobel/Prewitt operators. Then compute:

- Magnitude: |G| = √(GX²+GY²)  or approximation |GX|+|GY|

- Direction: α = arctan(GY/GX)

Result: two images — a gradient magnitude image and a gradient direction image. The magnitude image shows WHERE strong intensity changes are; the direction image shows WHICH WAY those changes point.

Canny Step 3 — Non-Maxima Suppression

Thinning the Thick Edge

After Step 2 we have a thick blob of high-gradient pixels. The goal of NMS is to produce ONE pixel wide edges — like a pen drawing.

Algorithm: For each pixel (x,y), look in the direction of the gradient (α direction). If this pixel's magnitude is NOT the maximum compared to its two neighbors in the gradient direction → suppress it (set to 0). If it IS the maximum → keep it.

This retains only the "ridge" of the gradient response — the single sharpest point across the edge band.

Why 4 Directions Is Enough

A continuous edge can run at any angle, but in a discrete pixel grid, we can only distinguish 4 meaningful directions. The gradient direction is a continuous value; quantizing to 4 bins doesn't cause much error because:

1. Most edges run predominantly in one of the 4 directions

2. Curved edges are locally straight (each pixel sees only its immediate neighborhood)

3. The hysteresis step in Step 4 reconnects any pixels that NMS may have incorrectly suppressed

Explain why the edge direction (perpendicular to gradient) is NOT what we check in NMS — we check along the GRADIENT direction, not the edge direction. Why would checking along the edge direction be wrong?
  • Edge direction = along the edge contour (e.g., if edge is a vertical line, edge direction = upward/downward).
  • Gradient direction = perpendicular to edge (e.g., for vertical edge, gradient points left or right).

  • Along the gradient direction: the edge is a ridge in gradient space. The pixel on the true edge has MAXIMUM gradient; pixels on either side (inside object, outside object) have lower gradient. So comparing along gradient direction correctly identifies the single maximum.
  • Think of the gradient as a mountain ridge: you check whether you're standing on the peak by looking perpendicular to the ridge (= along gradient), not along the ridge.

Canny Step 4 — Hysteresis Thresholding & Edge Linking

The Problem with Single Thresholding

After NMS we have thin edges, but there's still noise. A single threshold T: anything above T is an edge, anything below is not. Problem: if T too high → true edges with low contrast are missed (gaps in contours). If T too low → noise edges are included.

Solution: TWO thresholds TL (low) and TH (high). Recommended ratio TH:TL = 2:1 to 3:1.

Three categories of pixels:

1. M > TH → STRONG EDGE (definitely an edge)
2. TL ≤ M ≤ TH → WEAK EDGE (might be an edge)
3. M < TL → NOT AN EDGE (discard)

Edge linking rule: A WEAK edge pixel is kept as a true edge IF AND ONLY IF it is connected (directly or indirectly) to a STRONG edge pixel. Otherwise it is discarded.

This is called HYSTERESIS THRESHOLDING.

Double Thresholding — Hysteresis
Given: NMS-suppressed magnitude image M[x,y]
       Low threshold TL, High threshold TH  (TH:TL ≈ 2:1 to 3:1)
 
Classification:
  M > TH          → STRONG edge (always kept)
  TL ≤ M ≤ TH     → WEAK edge (kept only if connected to strong)
  M < TL          → NON-EDGE (discarded)
 
Edge linking (hysteresis):
  For each WEAK pixel:
    if any of its 8-neighbors is STRONG → promote to STRONG
    else → discard
  Repeat until no more changes (BFS/flood-fill)
 
Result: Only strong edges and their connected weak chains remain.

Conceptual Analogies That Make Edge Detection Click

Analogy 1 — The Cliff Metaphor

Imagine an image as a 3D terrain: pixel intensity = elevation. Flat regions are plains. Step edges are vertical cliff faces. Ramp edges are sloping hillsides. Roof edges are mountain ridges.

The gradient ∇F at each point is the "slope" of the terrain. An edge detector is a geologist mapping steep terrain — looking for anywhere the slope (gradient magnitude) exceeds a threshold.

Non-maxima suppression is the process of finding the single sharpest point on a slope, like finding the exact crest of a ridge from above.

Analogy 2 — The Marching Ant

Imagine an ant walking along the surface of an image. When the ant's forward direction is perpendicular to an edge, it climbs a steep hill (large gradient). When it walks parallel to an edge, the terrain is flat (small gradient in that direction).

Canny edge detection is the algorithm that makes the ant find and walk along the crest of every hill — ignoring side slopes (NMS) and deciding which hills are real mountains vs. noise bumps (hysteresis).

Analogy 3 — The Orchestra Tuner

The gradient operators (Roberts, Prewitt, Sobel) are like musical instruments tuned to different frequency ranges:

  • Roberts is a piccolo — sharp, precise, picks up high-frequency details but gets overwhelmed by noise.
  • Prewitt is a clarinet — balanced, general purpose.
  • Sobel is a cello — smoother, emphasises the stronger, lower-frequency edge signals.

Canny is the full orchestra conductor — uses all instruments in the right order, at the right time.

When Canny Fails — Research Evidence

Research

Five Documented Failure Modes of Canny Edge Detection

  • Low contrast edges are completely missed when contrast falls below noise floor
  • Texture surfaces flood edge maps with irrelevant gradient responses
  • Blur and defocus cause thick/merged edge responses dependent on σ choice
  • A single (TL, TH) pair cannot be optimal for all regions of a natural image
  • Cross-edge intersections produce rounded corners instead of sharp angles
Failure Mode 1 — Low Contrast Edges

Canny requires edges to be above threshold. In thermal imaging (night-vision cameras), edge contrast can be very low (2–5 pixel intensity difference vs. noise floor of ~10). Canny completely misses these.

Solution: Adaptive thresholding or deep learned features trained on thermal-domain data.

Failure Mode 2 — Texture vs. Edge Confusion

Highly textured surfaces (wood grain, fabric, grass) produce thousands of gradient responses. Canny detects all of them, flooding the edge map with irrelevant edges. No single threshold can separate texture from true object boundaries.

Solution: Semantic edge detection with per-category edge priors (e.g., CASENet, DDS).

Failure Mode 3 — Blur and Defocus

If σ is too small relative to blur, the pre-smoothing step is insufficient. If σ is too large, closely-spaced thin edges merge into one — they cannot be individually resolved.

Solution: Multi-scale edge detection — run Canny at σ=0.5, 1.0, 2.0 and combine results.

Failure Mode 4 — Threshold Sensitivity

A single pair (TL, TH) cannot be optimal for all regions of a natural image. Bright regions and dark regions have different noise characteristics — the same absolute threshold produces different false positive rates in different parts of the image.

Solution: Local adaptive thresholding — compute thresholds per-patch using local statistics (mean + k×std).

Failure Mode 5 — The Cross-Edge Problem

When two edges intersect at an angle, the gradient magnitude near the intersection is influenced by both edges simultaneously. Canny can miss the exact intersection or produce a "rounded corner" instead of a sharp angle.

Solution: Defective Edge Detection (arxiv 2411.14868, 2024) proposes a cascaded Canny ensemble specifically to address cross-edge failures at intersections.

Frequency Domain Intuition

The gradient operator (finite difference) is a HIGH-PASS FILTER in the frequency domain. It amplifies high-frequency components (edges, sharp transitions) and suppresses low-frequency components (smooth regions).

The Gaussian is a LOW-PASS FILTER — it suppresses high frequencies (noise).

Together: Gaussian (low-pass) then Gradient (high-pass) = BAND-PASS FILTER that passes edge-frequency content while suppressing both noise (too high) and flat regions (too low). In Fourier terms: applying ∂G/∂x to an image selects the frequency band corresponding to edge widths of σ pixels.

Frequency Domain Interpretation
Gaussian G(σ) → Low-pass filter  [suppresses f > 1/σ]
Gradient ∂/∂x → High-pass filter  [amplifies high frequencies]
Canny Step 1+2 = (∂G/∂x) * f → Band-pass filter  [edges at scale σ]
 
Increasing σ → shift band-pass to LOWER frequencies → detect coarser (wider) edges
Decreasing σ → shift band-pass to HIGHER frequencies → detect finer (thinner) edges
 
Practical rule: match σ to the expected edge width in your application

4 Medium Given: F[x,y]=100, F[x+1,y]=100, F[x,y+1]=250. Using forward difference, what is the gradient direction α = arctan(GY/GX)?
  • A 0°
  • B 45°
  • C 90°
  • D 135°
Correct Answer: C — 90°

Step-by-step computation:

  • GX = F[x+1,y] − F[x,y] = 100 − 100 = 0
  • GY = F[x,y+1] − F[x,y] = 250 − 100 = 150
  • α = arctan(GY / GX) = arctan(150/0) = arctan(∞) = 90°

8 Hard Prof explained the "hack" in Canny Step 1: rather than computing ∂/∂x(F * G) in two steps, we use ∂/∂x(F * G) = F * (∂G/∂x). Which mathematical property makes this true?
  • A The commutative property of convolution (F*G = G*F)
  • B The associative property of convolution ((A*B)*C = A*(B*C))
  • C The linearity (distributive property) of differentiation over convolution
  • D The Fourier shift theorem
Correct Answer: C — Linearity of differentiation

The key property is LINEARITY: differentiation is a linear operator, so it distributes over linear operations like convolution.

Formally: ∂/∂x(F * G) = F * (∂G/∂x)

  • A is wrong: Commutativity (F*G = G*F) governs the order of operands — not whether differentiation distributes.
  • B is wrong: Associativity (A*B)*C = A*(B*C) governs the grouping of multiple convolutions — not the interaction of differentiation with convolution.
  • D is wrong: The Fourier shift theorem describes how a shift in space corresponds to a phase change in frequency — unrelated here.

Signal processing perspective: Differentiation in the spatial domain = multiplication by jω in the frequency domain. Convolution in space = multiplication in frequency. So differentiating after convolving = multiplying by jω after multiplying spectra = same as pre-differentiating the kernel.

Common mistake: Saying "associativity." That governs how multiple convolutions can be reordered — it is a different (though related) algebraic property. The exact property here is the linearity of the differentiation operator distributing through the convolution.

  2. SPACE complexity increases — intermediate first-derivative values must be stored before the second derivative can be computed
