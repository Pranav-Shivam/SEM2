Image Pre-processing

What Pre-processing Actually Does foundation

Think of an image as a witness. Before you cross-examine it (extract features), it must be coherent — not distorted by noise, bad lighting, or scale mismatch. Pre-processing converts raw captured data into a form that algorithms can reliably analyse.

Every real-world image suffers from at least one of: illumination variance, sensor noise, scale mismatch, motion blur, or compression artifacts. Pre-processing attacks all of these before feature extraction.

Prof. Dhruba — Transcript
"If the features are good, then definitely you'll get a proper output. Without pre-processing, the noise may be treated as an important feature — that is also possible."

The canonical pipeline Prof. described: pre-process → extract features → post-process → output. This is the invariant structure of every CV system, from face recognition to tumour segmentation.

Pre-processing Pipeline ──────────────────────────────────── Raw Image → Noise Reduction (remove stochastic interference) → Normalisation (uniform scale, dtype, colour space) → Contrast Enhancement (amplify useful signal) → Filtering (sharpen edges or smooth regions) → Resizing (canonical spatial resolution) → Usable Feature Space
DL Bridge — OpenCV / PyTorch
Every CNN backbone has a fixed input resolution (224×224, 384×384). Resizing is therefore mandatory pre-processing for all DL inference. Normalising pixel values via transforms.Normalize(mean, std) in PyTorch is mathematically equivalent to contrast stretching at the data pipeline level.

Prof.'s face recognition example: two eyes and the mouth/nose-tip are the three anchor keypoints. Good pre-processing ensures these landmarks are distinguishable — illumination correction is the most critical step because shadow moves pixel values across detection thresholds.

Exam Alert
Questions ask why pre-processing is needed, not just what it does. Expected answer references: (1) noise treated as signal, (2) illumination variance, (3) scale mismatch preventing algorithm generalisation. The classic wrong answer is "to make the image look better" — that's a consequence, not the reason.

Generating Noise in Code notebook

Understanding noise mathematically requires knowing how each type is generated. From the webinar notebook — the three canonical implementations:

# 1. Gaussian Noise — ADDITIVE: independent of pixel value def add_gaussian_noise(image, mean=0, std=25): noise = np.random.normal(mean, std, image.shape).astype(np.float32) noisy = image.astype(np.float32) + noise # ADDITIVE return np.clip(noisy, 0, 255).astype(np.uint8) # 2. Salt & Pepper — random pixel corruption at extremes def add_salt_pepper_noise(image, salt_prob=0.01, pepper_prob=0.01): noisy = np.copy(image) total = image.shape[0] * image.shape[1] for _ in range(int(total * salt_prob)): i,j = random.randint(0,image.shape[0]-1), random.randint(0,image.shape[1]-1) noisy[i,j] = [255,255,255] # Salt = white for _ in range(int(total * pepper_prob)): i,j = random.randint(0,image.shape[0]-1), random.randint(0,image.shape[1]-1) noisy[i,j] = [0,0,0] # Pepper = black return noisy # 3. Speckle Noise — MULTIPLICATIVE: scales with pixel intensity def add_speckle_noise(image): noise = np.random.randn(*image.shape).astype(np.float32) noisy = image + image * noise # MULTIPLICATIVE return np.clip(noisy, 0, 255).astype(np.uint8)
Key Distinction — Additive vs Multiplicative Noise
Gaussian: noisy = image + noise — noise intensity is constant regardless of pixel value. A bright pixel (200) and dark pixel (20) get the same noise magnitude. Speckle: noisy = image + image * noise — noise scales proportionally. Pixel 200 gets 10× more noise than pixel 20. This is why speckle looks like granular texture concentrated in bright regions — it models coherent imaging physics (SAR, ultrasound, laser).

Q: Which filter is most effective for salt-and-pepper noise, and why?
Recall: salt-and-pepper noise is random isolated pixels at extreme values (0 or 255).
Averaging blur takes the mean — a single corrupt pixel (255) in a 3×3 neighbourhood raises the output from ~100 to ~128. The outlier is diluted, not eliminated.
Median filter sorts the 9 neighbourhood pixels and picks the middle value. A single 255 among 8 normal values (~90–110) falls at rank 9. The median (~100) completely discards the outlier.
→ Median filter. Statistically robust to outliers because it ignores values at extreme ranks.
Key insight: median preserves edges better than box blur because step edges appear as a rank transition, not an outlier.
Q: A 5×5 Gaussian kernel with σ=1 is applied. What happens to an edge pixel where left neighbours are all 50 and right neighbours are all 200?
Gaussian weights decay as exp(−(x²+y²)/2σ²). Centre weight is highest; periphery decays quickly.
The 5×5 window straddles the edge — roughly half the patch draws from the 50-region, half from the 200-region.
Weighted average: output pixel is between 50 and 200. Exact value depends on edge position in the window. The edge gradient is preserved but its magnitude is reduced.
→ Edge pixel averages toward ~125. Edge gradient is blurred (reduced magnitude) but not destroyed.
This is why Gaussian blur is used as a pre-step in Canny: it smooths noise while the edge gradient (though diminished) remains detectable.
Q: Derive why the bilateral filter preserves edges where Gaussian blur does not. Express the bilateral weight formally.
Standard Gaussian weight: w_s(x,y) = exp(−(x²+y²)/(2σ_s²)) — depends only on spatial distance.
Bilateral adds a range term: w_r(p,q) = exp(−|I_p−I_q|²/(2σ_r²)) — depends on intensity difference.
Combined: w(p,q) = w_s · w_r. Near an edge, pixels across it differ greatly in intensity → |I_p−I_q| is large → w_r ≈ 0 → those pixels are excluded from the average.
Result: only pixels on the same side of the edge (similar intensity) contribute. The filter smooths within homogeneous regions but does not average across discontinuities.
→ Bilateral = spatial Gaussian × range Gaussian. Range term kills cross-edge contributions — edge is preserved because high-contrast neighbours get near-zero weight.
σ_r controls edge sensitivity. Small σ_r = strong edge preservation, less smoothing. Large σ_r → approaches Gaussian blur. Prof. used 75 as a balanced default.
DL Bridge — Denoising
Denoising Autoencoders and Diffusion Models are the deep learning equivalents of bilateral/NL-Means. DDPM (Ho et al., 2020) explicitly frames image generation as a learnt reverse denoising process — noise is added in ~1000 forward steps and a U-Net learns to invert each step. NL-Means' non-local self-similarity principle directly inspired the Non-Local Means block in Video Transformer architectures.

Prof. Dhruba — Transcript (affine vs perspective)
"Affine transform and perspective transform are almost the same, but there's a little difference in it. So you just can explore affine transform later." [Prof. assigned as self-study. Key difference below.]

Affine vs Perspective: Affine uses 3 point pairs (2×3 matrix) and preserves parallel lines. Perspective uses 4 point pairs (3×3 homography) and can warp rectangles into arbitrary quadrilaterals — used for document rectification and bird's-eye view in autonomous driving.

Q: Write the translation matrix M that moves an image 100 pixels right and 50 pixels down.
Format: 2×3 matrix [[a,b,tx],[c,d,ty]] where a=d=1 (identity), b=c=0 (no rotation).
Right = positive x: tx=100. Down = positive y (image y increases downward): ty=50.
M = np.float32([[1, 0, 100], [0, 1, 50]])
Q: A 300×300 image is rotated 45°. Some pixels become black. Why, and how can this be mitigated?
Rotating a square 45° extends its corners to ±212px from the centre (300×√2/2 ≈ 212), exceeding the 150px half-width of the original bounding box.
Pixels that map outside the output canvas boundaries have no source pixel → filled with black (default borderMode=constant, value=0).
Mitigation: (a) expand output canvas to int(300√2)≈424 pixels, or (b) use borderMode=cv2.BORDER_REFLECT to fill from reflected border pixels.
→ Black corners appear because the rotated image corners fall outside the 300×300 output. Fix by enlarging the output canvas or using border reflection.
Q: Derive why bilinear interpolation is needed during scaling and compute the output for source position (2.3, 1.7).
Scaling maps output pixels back to non-integer source coordinates. Nearest-neighbour rounds to the closest pixel → blocky artefacts. Bilinear interpolates from the 4 surrounding integer-coordinate neighbours.
Source position (x,y)=(2.3, 1.7). Four neighbours: I₀₀=(2,1), I₁₀=(3,1), I₀₁=(2,2), I₁₁=(3,2). Fractional offsets: dx=0.3, dy=0.7.
Bilinear formula: output = (1−dx)(1−dy)·I₀₀ + dx(1−dy)·I₁₀ + (1−dx)dy·I₀₁ + dx·dy·I₁₁
Substituting: output = 0.7×0.3·I₀₀ + 0.3×0.3·I₁₀ + 0.7×0.7·I₀₁ + 0.3×0.7·I₁₁ = 0.21·I₀₀ + 0.09·I₁₀ + 0.49·I₀₁ + 0.21·I₁₁
→ output = 0.21·I₀₀ + 0.09·I₁₀ + 0.49·I₀₁ + 0.21·I₁₁ (weighted sum of 4 surrounding pixels)
cv2.INTER_AREA is best for downscaling (avoids aliasing). cv2.INTER_CUBIC uses 4×4 neighbours for smoother upscaling.
DL Bridge — Transforms → Data Augmentation
PyTorch's torchvision.transforms.RandomHorizontalFlip() and RandomAffine(degrees=15) ARE these geometric transforms applied stochastically at training time. Spatial Transformer Networks (Jaderberg et al., 2015) learn affine parameters end-to-end, making the network invariant to misalignment without manual pre-processing.
Image Arithmetic notebook

Before or after geometric transforms, images are often combined arithmetically. A critical distinction between OpenCV and NumPy:

# Image Arithmetic — from webinar notebook img1 = cv2.resize(img1, (300, 300)) img2 = cv2.resize(img2, (300, 300)) # must be same size # cv2.add — SATURATING (clips at 255) added = cv2.add(img1, img2) # 150 + 150 = 255 ✓ # numpy + — WRAPS AROUND (modular arithmetic) wrong = img1 + img2 # 150 + 150 = 44 ✗ (mod 256) # cv2.addWeighted — weighted blend (alpha compositing) blended = cv2.addWeighted(img1, 0.7, img2, 0.3, 0) # dst = 0.7*img1 + 0.3*img2 + 0 (gamma=0 bias) # Used for watermarking, overlay, double-exposure effects
Exam Alert — cv2.add vs numpy +
Classic trap: img1 + img2 in NumPy wraps mod 256 because uint8 arithmetic overflows. cv2.add(img1, img2) saturates at 255. For image processing, always use cv2.add. The common exam question: "A pixel value is 200 and you add 100. What is the output?" Answer: 255 (cv2) vs 44 (numpy). Use cv2.addWeighted for controlled blending with alpha channels.

Colour Space Conversions

Why Multiple Colour Spaces Exist colour theory

RGB is optimised for display hardware. It is poor for many CV tasks because illumination is coupled to colour — the same object under different lighting looks different in RGB but stays consistent in HSV. Different spaces decompose colour information optimally for different tasks.

Prof. Dhruba — Transcript
"Python will read all the images in the format of BGR. But for a normal human being, our eye will be taking the colour space in the format of red, green and blue. If we are not doing that [converting], then your entire image space and illumination will get changed."
SpaceChannelsWhy Use ItOpenCV Code
BGR→RGBBlue,Green,Red → R,G,BHuman display; Matplotlib requires RGBcv2.COLOR_BGR2RGB
GrayscaleL (luminance only)Single channel; edge detection; morphologycv2.COLOR_BGR2GRAY
HSVHue, Saturation, ValueColour segmentation regardless of brightnesscv2.COLOR_BGR2HSV
LABL* (lightness), A* (green↔red), B* (blue↔yellow)Perceptual uniformity; CLAHE on L onlycv2.COLOR_BGR2LAB
YCrCbLuma, Chroma-Red, Chroma-BlueSeparates brightness from colour for compressioncv2.COLOR_BGR2YCrCb

Prof. explained the LAB conversion path: BGR → XYZ (device-independent) → LAB. L=lightness, A=green-to-red axis, B=blue-to-yellow axis. It models human visual perception of colour differences — perceptually equal distances in LAB map to equal perceptual differences for humans. This is why CLAHE is typically applied to the L channel in LAB rather than directly to RGB.

# CLAHE on L-channel — perceptually correct approach lab = cv2.cvtColor(img, cv2.COLOR_BGR2LAB) l,a,b = cv2.split(lab) clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8,8)) l_eq = clahe.apply(l) result = cv2.cvtColor(cv2.merge([l_eq,a,b]), cv2.COLOR_LAB2BGR)
Exam Alert — BGR vs RGB Trap
Classic question: "Why does your image appear bluish when displayed with Matplotlib?" Answer: OpenCV reads BGR; Matplotlib expects RGB. Red and blue channels are swapped. Sky appears orange, skin looks teal. Always apply cv2.COLOR_BGR2RGB before plt.imshow().
HSV Colour Segmentation notebook

The most practical application of HSV: isolate objects by colour regardless of illumination. From the webinar notebook — detecting blue objects:

# HSV Color Segmentation — from Image_Processing_Demos notebook img = cv2.imread("image.jpg") hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV) # HSV ranges: H=[0,179], S=[0,255], V=[0,255] in OpenCV lower_blue = np.array([100, 150, 50]) # H~120°, min saturation, min value upper_blue = np.array([140, 255, 255]) # Binary mask: white=blue detected, black=not blue mask = cv2.inRange(hsv, lower_blue, upper_blue) # Extract only blue regions (AND with original) result = cv2.bitwise_and(img, img, mask=mask) # Common HSV ranges (OpenCV H is 0-179, i.e. degrees/2): # Red: [0,120,70] → [10,255,255] (also [170,120,70] → [180,255,255]) # Green: [36,50,70] → [86,255,255] # Blue: [100,150,50] → [140,255,255] # Yellow: [20,100,100] → [30,255,255]
Why HSV Beats RGB for Segmentation
In RGB: a blue object under bright light has high R, G, B values. The same blue object in shadow has low R, G, B. These look completely different — a threshold on any RGB channel will miss one. In HSV: both cases share the same Hue (~120°). V (brightness) varies; H stays constant. cv2.inRange on H alone separates colours robustly across illumination conditions. This is why HSV is the standard space for colour-based object detection and tracking.

Morphological Operations

Shape-Based Image Processing binary ops

Morphological operations process images based on shape. They require a binary image (white=foreground=1, black=background=0) and a structuring element (kernel). The kernel probes the image — its shape and size determine the operation's behaviour.

Prof. Dhruba — Transcript
"The kernel slides through the image. Normally, the kernel will be of odd number size. The reason for that is you are getting the information about one particular pixel — the centre pixel is the attractor or the exact value that you are looking for."

Odd-sized kernels (3×3, 5×5, 7×7) are required: only odd dimensions have a unique centre pixel. Even kernels have no unambiguous centre.

# Create structuring element kernel = np.ones((3, 3), np.uint8) eroded = cv2.erode(img, kernel, iterations=1) dilated = cv2.dilate(img, kernel) opened = cv2.morphologyEx(img, cv2.MORPH_OPEN, kernel) closed = cv2.morphologyEx(img, cv2.MORPH_CLOSE, kernel) gradient = cv2.morphologyEx(img, cv2.MORPH_GRADIENT, kernel) tophat = cv2.morphologyEx(img, cv2.MORPH_TOPHAT, kernel) blackhat = cv2.morphologyEx(img, cv2.MORPH_BLACKHAT, kernel)
Q: After erosion a binary image has fewer white pixels; after dilation it has more. What does Opening preserve?
Erosion removes small white regions and thins all white regions.
Subsequent dilation restores the size of white regions that survived erosion — only the large ones, because small ones were completely removed.
→ Opening preserves large white objects while removing small noise blobs. Objects larger than the kernel survive; smaller ones are erased.
Q: A binary text image has a small gap in one stroke. Which operation closes it, and why does it work?
The gap = dark pixels within what should be a white stroke — a small hole in a white region.
Closing = dilation then erosion. Dilation expands white regions in all directions. If the gap is smaller than the kernel, expansion from both sides meets and fills it.
Erosion then shrinks all white regions back — but the filled gap, now part of the larger white region, is too large to be removed by erosion.
→ Closing fills small gaps. cv2.morphologyEx(img, cv2.MORPH_CLOSE, kernel). Kernel size must be ≥ the gap width.
Q: Derive what the Morphological Gradient represents and why it produces object outlines.
Dilation(f) ⊇ f ⊇ Erosion(f) for any binary image. Dilation expands each white blob outward by one kernel radius; erosion shrinks it inward.
At interior pixels: both return 1. Gradient = 1−1 = 0. Interior is black.
At boundary pixels: dilation returns 1 (expanded outward), erosion returns 0 (shrank inward past the boundary). Gradient = 1−0 = 1. Boundary is white.
At exterior pixels: both return 0. Gradient = 0. Exterior is black.
→ Gradient = 1 only at object boundaries. Interior and exterior cancel to 0. This is a morphological edge detector operating on binary masks.
Compared to Canny (greyscale), morphological gradient works on binary masks — useful post-thresholding for finding exact object outlines.
Research Insight — Morphology in Deep Learning (2025)

[Arxiv 2505.09710, 2025] Deep Morphological Neural Networks function as universal approximators. Key insight: max-pooling in CNNs is mathematically equivalent to morphological dilation with a square structuring element. Learnable morphological layers allow networks to learn arbitrary structuring element shapes, outperforming fixed max-pooling on boundary detection tasks with only 0.01% of competing methods' parameter count.

Histogram Equalisation & CLAHE

Contrast Enhancement via Intensity Redistribution histogram

A histogram shows how many pixels fall at each intensity level (0–255). A low-contrast image has a narrow, concentrated histogram. Before equalisation, contrast stretching provides a simpler linear fix.

Contrast Stretching (Linear Normalisation) notebook

If an image only uses intensities 80–180 (narrow range), stretching linearly maps 80→0 and 180→255, filling the full dynamic range without changing the shape of the distribution.

# Contrast Stretching — linear normalisation # Formula: I' = (I - Imin) / (Imax - Imin) × 255 img = cv2.imread("image.jpg", 0) # grayscale Imin = img.min() Imax = img.max() norm = ((img - Imin) / (Imax - Imin) * 255).astype(np.uint8) # Equivalent OpenCV call: norm2 = cv2.normalize(img, None, 0, 255, cv2.NORM_MINMAX)
Contrast Stretching vs Histogram Equalisation
Stretching is LINEAR — maps range [Imin,Imax] to [0,255]. Preserves the relative distribution (histogram shape shifts and scales, not warped). Fast, no side effects. Best when the image simply underutilises the full range (e.g., 8-bit sensor using only 80–180). HE is NON-LINEAR — remaps each intensity via CDF. Forces a flat/uniform histogram. More aggressive; can invert shadows in night-time images. Use HE when you need maximum contrast enhancement; use stretching when you want to correct range only.

A histogram shows how many pixels fall at each intensity level (0–255). A low-contrast image has a narrow, concentrated histogram. Equalisation spreads this distribution across the full 0–255 range, making features easier to detect.

# Histogram Equalisation — 3-step algorithm Step 1: Compute histogram h(v) = number of pixels with intensity v (v ∈ [0,255]) Step 2: Compute CDF CDF(v) = Σ h(k) for k = 0 to v Step 3: Normalise CDF to [0,255] → new intensity mapping new_v = round[ (CDF(v) − CDF_min) / (N − CDF_min) × 255 ] where N = total pixel count equ = cv2.equalizeHist(gray_img) # applies to single channel

Prof.'s description: "How red, green and blue colours are being distributed in the whole image — with respect to that you get the histogram. Using that, compute the cumulative distribution function and compute the new normalised value and replace the pixel values."

Problem with Basic Equalisation
Global HE treats the entire image as one distribution. In an image with a bright window region and a dark interior, global equalisation over-amplifies the dark region or washes out the bright one — it cannot adapt locally.

CLAHE solves this: Contrast Limited Adaptive Histogram Equalisation.

# CLAHE Algorithm — Prof. Dhruba's exact description Step 1: Divide image into tiles (e.g. 8×8 grid of patches) Step 2: Compute histogram for each tile independently Step 3: CLIP histogram at clipLimit (redistribute excess to all bins) if h(v) > clipLimit: redistribute excess evenly Step 4: Equalise the clipped histogram for each tile Step 5: Bilinear blend at tile boundaries → smooth transitions clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8,8)) result = clahe.apply(channel)

Research Insight — CLAHE as Domain Adaptation (2024)

[Yoshimi et al., Oral Surgery 2024] CLAHE normalises scanner-specific contrast biases in MRI images — models trained on CLAHE-preprocessed data from one MRI scanner generalise significantly better to a different scanner. This reframes CLAHE from "contrast enhancement technique" to a domain alignment step that reduces distribution shift between acquisition devices. Same principle applies to autonomous vehicle cameras — CLAHE normalises cross-sensor illumination variance before the perception model runs. [PubMed 2024] CLAHE + transfer learning + augmentation achieved F1=0.89 on brain image classification, outperforming all other preprocessing combinations.

Exam Alert — CLAHE vs Basic HE
Three key differences: (1) CLAHE operates tile-by-tile, not globally; (2) CLAHE clips the histogram at clipLimit before equalising, preventing noise amplification; (3) CLAHE applies bilinear blending at tile boundaries to prevent blocking artefacts. Common mistake: saying "CLAHE is just HE applied multiple times" — wrong. Clipping and tile-based processing are the fundamental innovations.

Image Sharpening

Amplifying High-Frequency Detail filtering

Sharpening is the inverse goal of denoising — instead of suppressing high-frequency content, we amplify it. Edges and fine detail are high-frequency; blurring removes them; sharpening adds them back. The key tool is the Unsharp Mask.

Intuition — Why "Unsharp" Mask?
Paradoxically named from darkroom photography: a blurred ("unsharp") copy of the negative was used as a mask to reveal detail. Digitally: blur the image, subtract the blur from the original to extract edges, add those edges back. The "mask" is the blur — you use it to find what to enhance.
# Unsharp Mask — from Image_Processing_Demos notebook # Algorithm: Sharp = Original + k × (Original − Blurred) # = Original + k × (high-frequency detail) blur = cv2.GaussianBlur(img, (0,0), sigmaX=3) # addWeighted: dst = alpha×src1 + beta×src2 + gamma sharp = cv2.addWeighted(img, 1.5, blur, -0.5, 0) # = 1.5×orig - 0.5×blur # = orig + 0.5×(orig - blur) # = orig + 0.5×edges ← amplifies edge frequencies # Kernel-based alternative (equivalent for 3×3): kernel = np.array([[0,-1,0],[-1,5,-1],[0,-1,0]]) sharp2 = cv2.filter2D(img, -1, kernel) # Centre weight 5 amplifies pixel; neighbours −1 subtract surroundings # Net: pixel gets 5×itself − 4 neighbours = pixel + (pixel − mean_neighbours)
Q: What does unsharp masking amplify, and why does blurring first help?
Blurring removes high-frequency content (edges, fine detail), leaving only low-frequency (smooth regions).
Original − Blurred = Original − low-frequency = high-frequency detail only (an edge map).
Adding that edge map back to the original amplifies the edges relative to smooth regions → sharpening.
→ Amplifies high-frequency edge detail. Blurring first isolates those frequencies by subtraction.
Q: cv2.addWeighted(img, 1.5, blur, -0.5, 0) — derive what this computes algebraically.
Formula: dst = α·src1 + β·src2 + γ = 1.5·img + (−0.5)·blur + 0
= img + 0.5·img − 0.5·blur = img + 0.5·(img − blur)
= img + 0.5·edges. This is Unsharp Mask with strength k=0.5.
→ Unsharp Mask with k=0.5: adds 50% of the edge signal back to the original. Increasing α (e.g. 2.0, β=-1.0) increases sharpening strength.
Clipping: addWeighted clips to [0,255]. Strong sharpening near bright edges creates halos (bright band above threshold). Use small k for realistic results.
Q: When does sharpening amplify noise and how to mitigate it?
Noise is high-frequency — small, random, spatially isolated intensity variations. Sharpening amplifies ALL high-frequency content: both edges (wanted) and noise (unwanted).
The Gaussian blur step does not perfectly separate noise from edges: low-σ blur leaves noise in the residual; high-σ blur blurs true edges alongside noise.
Mitigation strategies: (1) denoise BEFORE sharpening (bilateral filter preserves edges, removes noise — then sharpen the clean image); (2) use edge-aware sharpening that only applies in high-gradient regions (detected via Canny mask); (3) reduce sharpening strength k.
In practice: always denoise → then sharpen. The order matters — sharpening a noisy image amplifies artefacts irreversibly.
→ Sharpening amplifies noise because noise is high-frequency. Fix: bilateral denoise first, then unsharp mask. Or constrain sharpening to detected edge regions only.
DL Bridge — Sharpening as High-Pass Filtering
The sharpening kernel [[0,-1,0],[-1,5,-1],[0,-1,0]] is a high-pass filter — it passes high frequencies and suppresses low. CNN kernels at layer 1 learn oriented high-pass filters spontaneously from data (Gabor-like). The Laplacian of Gaussian (LoG = blur then Laplacian) is the theoretical basis for SIFT's Difference-of-Gaussian blob detection. Modern image super-resolution models (ESRGAN, Real-ESRGAN) learn to infer high-frequency detail that was never in the low-res input — generalised learned unsharp masking.

Thresholding converts a grayscale image to binary (0 or 255) — separating foreground from background. It is the simplest segmentation technique and feeds directly into morphological operations, contour detection, and feature extraction.

Q: Why does global thresholding fail on a document image with a bright window on one side and a shadow on the other?
Global T is applied uniformly. Under the shadow, text pixels have intensity ~100; background ~60. A threshold of T=127 would classify both as background (both < 127) — text in shadow is invisible.
Under the bright window, background pixels might be 200; text 160. T=127 makes both foreground (both > 127) — pure white page looks like all text.
Adaptive thresholding computes a local T for each pixel based on its neighbourhood. In the shadow region, T≈80 (correctly separates text=100 from bg=60). In the bright region, T≈180.
→ Use adaptive thresholding for non-uniform illumination. blockSize controls neighbourhood; C adjusts sensitivity.
Q: Otsu's method requires a bimodal histogram. What happens if applied to an image with three intensity clusters?
Otsu assumes two classes (foreground/background). With three clusters (e.g. dark/grey/white), it finds the threshold that best separates two groups — merging the two closest clusters on one side.
The between-class variance σ²_b is still maximised — but that maximum may not correspond to a meaningful semantic boundary. One pair of adjacent clusters gets merged.
Fix: Multi-Otsu (three classes, two thresholds) or k-means clustering on pixel intensities (k=3).
→ Single Otsu merges two of three clusters. Use Multi-Otsu or k-means for multimodal histograms.
Q: Derive why Otsu's within-class variance minimisation is equivalent to between-class variance maximisation.
Total variance: σ²_total = w₀·σ₀² + w₁·σ₁² + w₀·w₁·(μ₀−μ₁)² = σ²_w + σ²_b. This is the law of total variance.
σ²_total is fixed (it's a property of the image histogram, independent of T).
Since σ²_total = σ²_w + σ²_b = constant, minimising σ²_w is identical to maximising σ²_b. They are mathematically equivalent optimisation problems.
σ²_b = w₀·w₁·(μ₀−μ₁)² is cheaper to compute: iterate over all possible T values, compute w and μ for each class from cumulative histogram — O(256) operations.
→ Total variance is constant. Minimising within = maximising between. Otsu solves efficiently via O(256) sweep over cumulative histogram.
Implementation: compute CDF once (256 values). For each T: w₀=CDF(T), w₁=1−CDF(T), μ₀/μ₁ from partial sums. Evaluate σ²_b. O(256) total.
DL Bridge — Thresholding → Sigmoid/Softmax
Global thresholding is a hard decision boundary at T. Neural network classification is soft thresholding: the sigmoid function σ(x) = 1/(1+e⁻ˣ) produces a smooth, differentiable approximation to the step function. Adaptive thresholding → spatial attention mechanisms: both modulate the decision threshold based on local context rather than applying a global value. Otsu's between-class variance maximisation is directly related to the Fisher Linear Discriminant — maximise class separation, the foundation of LDA.

Contour Detection & Colour Segmentation

Finding and Measuring Object Boundaries contours

Contours are curves joining all continuous points along a boundary with the same colour or intensity. They are extracted from binary images and enable object counting, shape analysis, region extraction, and area/perimeter measurement.

# Contour Detection Pipeline — from Image_Processing_Demos notebook img = cv2.imread("image.jpg") gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY) # Step 1: Binarise (Otsu gives clean binary for contour extraction) _, binary = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU) # Step 2: Find contours (Suzuki's border-following algorithm) contours, hierarchy = cv2.findContours( binary, cv2.RETR_EXTERNAL, # only outermost contours cv2.CHAIN_APPROX_SIMPLE # compress straight segments ) # Step 3: Draw all contours out = img.copy() cv2.drawContours(out, contours, -1, (0,255,0), 2) # -1 = draw all; (0,255,0)=green; 2=thickness in px # Step 4: Analyse each contour for cnt in contours: area = cv2.contourArea(cnt) # pixels² perimeter = cv2.arcLength(cnt, closed=True) # pixels x,y,w,h = cv2.boundingRect(cnt) # bounding box M = cv2.moments(cnt) # shape moments if M["m00"] != 0: # centroid cx = int(M["m10"] / M["m00"]) cy = int(M["m01"] / M["m00"])

Research Deep Dive

The Denoising–Generation Duality

Score-based diffusion models (Song & Ermon, 2020; Ho et al., 2020) frame image generation as a learned reverse diffusion — iterative denoising from pure Gaussian noise. Each denoising step is a neural approximation of the Gaussian filter Prof. described, but conditioned on semantic content. Every Stable Diffusion image is the result of applying ~1000 learnt denoising filters to a noise image.

DDPM (Ho et al., NeurIPS 2020); Score-Based Generative Models (Song et al., NeurIPS 2020)

Morphological Operations as Neural Primitives

Max-pooling in every ResNet, VGG, and ViT is provably equivalent to morphological dilation with a flat square structuring element. Research 2024–2025 shows learnable morphological layers achieve better boundary detection at 0.01% of the parameter count — because the learned structuring element adapts to actual data shapes rather than assuming squares.

Deep Morphological Networks (Franchi et al., 2020); Universal Approximation for Deep Morphological Networks (Arxiv 2505.09710, 2025)

When Classical Algorithms Fail

Canny fails on textures with high-frequency content at similar intensity to edges (grass, fabric) — generates false detections throughout. Fix: texture-aware segmentation with SLIC superpixels before Canny. Median filter destroys fine details (thin lines, small text) when the kernel is large relative to the feature. Harris corners are not scale-invariant — a corner at 8× downsampling may not register. Fix: Harris in a scale-space pyramid (what SIFT builds). Histogram equalisation inverts shadow regions in night-time imagery — CLAHE's clipping mitigates this.

HardBilateral Filter
8. A bilateral filter with sigmaColor=5 (vs Prof.'s default of 75) is applied. What is the effect on edges and noise removal?

  • B. More noise removal but blurred edges

HardMorphological Gradient
9. The Morphological Gradient = Dilation(f) − Erosion(f). What does it represent, and how does it relate to Sobel?

11. Otsu's method selects threshold T* by optimising which criterion?

EasySharpening
12. Unsharp masking uses cv2.addWeighted(img, 1.5, blur, -0.5, 0). What does this algebraically compute?
  • A. img − 0.5 × blur (simple subtraction)
  • B. img + 0.5 × (img − blur) — original plus half the edge residual
  • C. 1.5 × img − 0.5 × img (scales without using blur)
  • D. A Laplacian of the blurred image
Correct: B. dst = 1.5×img + (−0.5)×blur = img + 0.5×img − 0.5×blur = img + 0.5×(img − blur). The term (img − blur) is the high-frequency residual (edge detail). Adding 0.5 of it back amplifies edges — this is Unsharp Mask with k=0.5.
