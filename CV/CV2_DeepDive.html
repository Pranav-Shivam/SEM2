Section 01

What is an image?

An image is simultaneously a physical phenomenon, a mathematical function, and a computational matrix. These three views are equivalent — and each one is used in different parts of the CV pipeline.

Three equivalent representations
Physical
Light → sensor
Reflected EM energy captured by an A2D imaging system
Mathematical
f : ℝ² → ℝ
f(x,y) gives intensity (scalar) at each coordinate
Computational
M × N matrix
Grid of integers, 0 = black, 2ⁿ−1 = white
The grayscale intensity modelkey formula
ℓ = f(x, y) ← intensity at pixel (x,y) Constraint: L_min ≤ ℓ ≤ L_max L_min = i_min × r_min L_max = i_max × r_max For 8-bit system: L_min = 0, L_max = 255 (= 2⁸ − 1) Grayscale shape: M × N × 1 RGB shape: M × N × 3 (3 channels, each 0–255)

The intensity value ℓ is a scalar quantity. It is proportional to the energy radiated by the electromagnetic source at that point (slide 16). The physical meaning depends on the imaging modality — visible light, infrared, X-ray, etc.

Question

A 128×128 RGB image is loaded. Calculate: (a) total number of pixels, (b) total bits required to store it uncompressed at 8-bit depth, (c) memory in MB, (d) center pixel coordinates.

2
Solution
(a) Total pixels: 128 × 128 = 16,384 pixels (b) Storage (8-bit per channel, 3 channels): 128 × 128 × 3 × 8 bits = 393,216 bits (c) Memory: 393,216 ÷ 8 = 49,152 bytes = 48 KB = 0.047 MB (d) Center pixel: (⌊128/2⌋, ⌊128/2⌋) = (64, 64)
3

1
Question

An MRI scanner produces floating-point images with intensity values ranging from 0.0 to 4,500.0. You need to store this as an 8-bit grayscale image for a CV pipeline. (a) What process is this? (b) Derive the mapping formula. (c) Pixel with MRI value 2250.0 maps to what 8-bit value? (d) What information is lost and why?

2
Solution
(a) Process: QUANTIZATION — reducing intensity resolution from continuous/high-bit to 8-bit discrete levels. (b) Mapping (linear quantization / contrast stretching): output = round( (input - min) / (max - min) × (L-1) ) output = round( (input - 0) / (4500 - 0) × 255 ) output = round( input × 255 / 4500 ) (c) For input = 2250.0: output = round(2250 × 255 / 4500) = round(127.5) = 128 (d) Information lost: Original has 4500 distinct values. 8-bit has only 256 levels. Quantization step size = 4500 / 255 ≈ 17.6 → Values within ±8.8 of each other map to same pixel. This is LOSSY — fine intensity differences are merged. Solution: use 12-bit or 16-bit output instead.
Section 02

Camera &
image formation

How physics translates a scene into a pixel matrix — including the complete two-component intensity model from slide 16 of CV_CS_2.

The image formation equationmust know
f(x, y) = i(x, y) × r(x, y) i(x,y) = illumination component 0 ≤ i(x,y) < ∞ r(x,y) = reflectance component 0 ≤ r(x,y) ≤ 1 r = 0 → total absorption (Vantablack — annotated in slide 16) r = 1 → total reflectance (mirror / perfect glass) 0 < r < 1 → partial reflectance (all real-world surfaces)

The physical meaning of f(x,y) is determined by the source of the image, and its values are proportional to energy radiated by a physical source — electromagnetic waves (slide 16). For indoor light sources, typical i(x,y) values range from 0 (night) to 9×10⁴ lux (direct sunlight).

Pinhole vs lens
Pinhole camera
  • Single light beam per moment
  • Image inverted on back screen
  • Very long exposure needed
  • Simple but impractical at speed
Lens camera
  • Gathers light from many angles
  • Reduces exposure time drastically
  • Enables zoom & focal length control
  • Used in all modern CV systems

"Why do we use lenses? They gather more light than pinhole cameras, reducing exposure time. A pinhole camera allows only a single beam of light to pass at one particular instance in time."

Prof. Dhruba · May 2 2026 · Slide 15 quiz answer

1
Question

A pixel has illumination i(x,y) = 0.90 and reflectance r(x,y) = 0.80 (normalised units). Calculate f(x,y). Is this closer to black or white?

2
Solution
f(x,y) = i(x,y) × r(x,y) = 0.90 × 0.80 = 0.72 Scaled to 8-bit: 0.72 × 255 ≈ 184 Since 184 > 127 (midpoint), this pixel is closer to WHITE.
1
Question

The same scene is photographed outdoors (i=0.95) and indoors (i=0.30). The surface reflectance r(x,y) = 0.60 at both locations. What are the two pixel values? What is the ratio? What does this tell us about illumination-invariant recognition?

2
Solution
Outdoor: f = 0.95 × 0.60 = 0.570 → 8-bit: 145 Indoor: f = 0.30 × 0.60 = 0.180 → 8-bit: 46 Ratio: 145 / 46 ≈ 3.15 — same surface, ~3× different pixel value! Implication: A raw pixel value does NOT uniquely identify a surface. This is why deep learning models need illumination augmentation (brightness shifts, lighting changes) during training — to learn features of r(x,y) independently of i(x,y).
1
Question

Vantablack has r = 0.0035 (absorbs 99.965% of light). A standard white surface has r = 0.98. Under the same illumination i(x,y) = 1000 lux, calculate both pixel values (normalise lux to [0,1] using max of 10,000 lux). Then, if you apply the log transform s = c·log(1+r) with c = 255/log(2), what are the output values? Why is the log transform useful here?

2
Solution
Normalised i = 1000/10000 = 0.1 Vantablack: f = 0.1 × 0.0035 = 0.00035 → 8-bit: 0.089 ≈ 0 White surface: f = 0.1 × 0.98 = 0.098 → 8-bit: 24.99 ≈ 25 Without log transform: both look nearly black (0 vs 25)! The 25:1 reflectance ratio is lost in the dark range 0–25. With log transform (c = 255/log(2) ≈ 367.9): Vantablack: s = 367.9 × log(1+0.00035) ≈ 367.9 × 0.000350 ≈ 0.13 ≈ 0 White: s = 367.9 × log(1+0.098) ≈ 367.9 × 0.0936 ≈ 34.4 ≈ 34 Still compressed — but this is the same image in dim light. Under full illumination (i=1.0) the distinction becomes clear. Key insight: Log transform expands the dark range and compresses the bright range, revealing fine details in low-intensity regions. This is exactly why it is used on Fourier spectrum images (slide 31).
Section 03

Sampling &
quantization

The two-step conversion from a continuous scene to a discrete digital image. Every resolution and format decision in CV and deep learning is a consequence of these two operations.

The two steps
Sampling = spatial discretization

Convert the continuous scene into a fixed M×N grid. Each cell in the grid is one pixel. The scene size does not matter — it always maps to M×N cells.

Quantization = intensity discretization

Assign each pixel a discrete integer intensity. An n-bit system has 2ⁿ levels, ranging from 0 to 2ⁿ−1. The finer the quantization, the more intensity gradations you can represent.

# Intensity range for n-bit system: Range = [0, 2ⁿ − 1] # Common values (from slide 14 annotations): 1-bit → 2 levels (0–1) ← binary image 3-bit → 8 levels (0–7) ← older ADAS systems 8-bit → 256 levels (0–255) ← standard camera 12-bit → 4096 levels (0–4095) ← RAW format cameras 16-bit → 65536 levels ← MRI, CT scanners

A continuous greyscale image is sampled at 8×8 with 4-bit quantization. (a) How many pixels? (b) How many intensity levels? (c) If the original scene had dimensions 2m×2m, what is the spatial sampling interval in cm? (d) What YouTube resolution is 8×8 most analogous to, and what would 720p be?

1
Question

A 1280×720 image (8-bit, grayscale) is to be stored. (a) Memory needed uncompressed. (b) You reduce sampling to 640×360 — what fraction of original data? (c) You reduce quantization from 8-bit to 4-bit — what fraction? (d) Combined reduction? (e) If deep learning requires 224×224×3 input, how many bytes? (f) For a batch of 32 such images, how much GPU memory?

2
Solution
(a) Original: 1280 × 720 × 1 byte = 921,600 bytes ≈ 0.88 MB (b) Reduced sampling (640×360): 640 × 360 = 230,400 bytes Fraction = 230,400/921,600 = 1/4 = 25% (c) Reduced quantization (4-bit vs 8-bit): 4-bit = 0.5 bytes/pixel Fraction = 0.5/1.0 = 1/2 = 50% (d) Combined reduction: 1/4 × 1/2 = 1/8 = 12.5% of original data (e) DL input: 224 × 224 × 3 × 1 byte = 150,528 bytes ≈ 147 KB (f) Batch of 32: 32 × 150,528 = 4,816,896 bytes ≈ 4.59 MB (Plus model weights, gradients, activations — GPU needs much more)
Section 04

Spatial & intensity
resolution

Two independent dimensions of image quality — spatial resolution (how many pixels) and intensity resolution (how many grey levels). Both come directly from slides 21–23 of CV_CS_2.

Spatial resolution — smallest discernible detail

Spatial resolution is measured in DPI (dots per inch) — the number of sample points per inch of the original physical scene. Higher DPI = finer sampling = sharper image.

DPITypical useVisual qualitySlide 21 annotation
72 DPIScreen display (old web standard)Heavily pixelated on printWatch face unreadable
150 DPINewspaper printPixelation visible at edgesArm details blurry
300 DPIStandard laser printerAcceptable, slight blur60 slightly unclear
930 DPIHigh-end printing, slidesCrisp, every detail visibleAll tick marks clear
Intensity resolution — smallest discernible brightness change

Controlled by bit depth. The slides (22–23) show a bottle image at decreasing intensity levels from 256 to 2. Key observations from CV_CS_2:

LevelsBit depthVisual effect (slide 22–23)
2568-bitFull photographic quality, smooth gradients
1287-bitNearly identical to 256, minor loss in shadows
646-bitGrayish area at top starts to vanish
325-bitClearly missing mid-tones, losing detail
164-bitHalos (false contouring) appear visibly
42-bitStrong bands, only 4 distinct shades
21-bitBinary — only black and white, all gradients lost
False contouring

When intensity levels drop too low (typically below 32 for natural images), false contours / halos appear — visible bands at boundaries between intensity regions. This artifact is a key diagnostic sign of insufficient quantization.

Easy

1
Question

A 4-bit image (16 levels) is upsampled to 8-bit (256 levels). How many new intensity levels are added? Does this improve actual image quality?

2
Solution
New levels = 256 - 16 = 240 new levels added. Does it improve quality? NO — this is upsampling without new information. The original only contained 16 distinct intensities. Interpolating to 256 levels does not recover lost detail. It only spreads existing values wider apart. Analogy: stretching a 16-pixel image to 256 pixels does not add new scene detail — same reason.
1
Question

An image has spatial resolution 300 DPI and intensity resolution 8-bit. You need to print it at 150 DPI. (a) What happens to the pixel count? (b) What happens to visible quality? (c) Can you increase intensity resolution to compensate for lower spatial resolution?

2
Solution
(a) Pixel count at 150 DPI = (150/300)² × original = 0.25× fewer pixels (1/4 of original) (b) Quality: spatial sharpness reduces — edges appear blurrier, fine text loses legibility. The watch face example from slide 21: at 150 DPI some tick marks blur. (c) No — spatial and intensity resolution are INDEPENDENT. Adding more grey levels (e.g., 16-bit) does not recover spatial detail. Each dimension must be set appropriately. A 16-bit 72 DPI image is sharp in colour but spatially blurry. A 1-bit 930 DPI image is sharp spatially but only B&W.
1
Question

Derive the relationship between storage size, spatial resolution (M×N pixels), and intensity resolution (k bits). Then answer: (a) A 512×512 image at 8 bits. (b) Same image at 4 bits — storage? (c) A 256×256 image at 8 bits — same storage as which of the above? (d) What general relationship exists?

2
Solution
Storage = M × N × k (in bits) (a) 512×512, 8-bit: = 512 × 512 × 8 = 2,097,152 bits = 256 KB (b) 512×512, 4-bit: = 512 × 512 × 4 = 1,048,576 bits = 128 KB → Exactly half of (a) (c) 256×256, 8-bit: = 256 × 256 × 8 = 524,288 bits = 64 KB → 1/4 of (a) (d) General relationship: To keep storage constant when halving spatial resolution: You can DOUBLE the bit depth. 512×512 @ 4-bit = 1 MB 1024×1024 @ 1-bit = 1 MB (same storage, more spatial detail but only binary intensity — very different visual result!) Trade-off formula: M×N×k = constant Decreasing M,N by factor f requires increasing k by f².
Section 05

Distance metrics &
neighborhoods

Three distance measures and two types of pixel neighborhoods — directly from slide 24–25 of CV_CS_2. These concepts recur in every lab, from edge detection to segmentation.

The three distance measuresexam — expect numericals
For pixels p = (i, j) and q = (h, k): D_E(p,q) = √[(i−h)² + (j−k)²] ← Euclidean (L2 norm) — straight line D_4(p,q) = |i−h| + |j−k| ← City-block (L1 norm) — Manhattan D_8(p,q) = max(|i−h|, |j−k|) ← Chessboard (L∞ norm) # Norm equivalence (from image processing review notes): D_E ↔ L2 D_4 ↔ L1 D_8 ↔ L∞
Key property from slides

D_4 and D_8 are path-independent — they depend only on the endpoint coordinates, not on the actual path between them. This makes them computationally cheap. D_E requires a square root and is computationally expensive for large grids.

In a 5×5 image, the center pixel P is at (2,2). (a) List ALL pixels that satisfy D_4 ≤ 2 (diamond contour). (b) List ALL pixels that satisfy D_8 ≤ 2 (square contour). (c) How many pixels are in each set? (d) What shape do these contours form and why does this matter for convolution operations?

Section 06

Image histogram

The histogram summarises the complete intensity distribution of an image in one compact structure. Slide 29 of CV_CS_2 defines both forms. It is the foundation for contrast stretching and histogram equalisation.

Histogram definitionsslide 29
# Un-normalized histogram (raw counts): h(r_k) = n_k for k = 0, 1, 2, ..., L−1 # n_k = number of pixels in image f with intensity r_k # Normalized histogram (probability distribution): p(r_k) = h(r_k) / MN = n_k / MN # M, N = number of image rows and columns # Property: Σ p(r_k) for all k = 1.0 (always) # We mostly work with normalized histograms (slide 29 note)
Why normalized?

The normalized histogram is a probability mass function (PMF) — it tells you the probability that a randomly selected pixel has intensity r_k. This property makes it usable across images of different sizes, and is directly used in contrast stretching and histogram equalisation formulas.

Question

A 4×4, 3-bit image (L=8) has values: [[1,1,2,2],[1,2,3,3],[2,3,4,4],[3,4,5,5]]. (a) Compute normalized histogram. (b) Compute cumulative distribution. (c) Apply contrast stretching (output 0–7). (d) What shape does the histogram suggest about image contrast?

Q1
An 8-bit grayscale image has pixel value 180 at position (3,4). Apply the negative transform and compute the output. Is the output pixel lighter or darker?

Q3

Q8
COMPREHENSIVE: An 8-bit image has f(x,y) = i(x,y) × r(x,y). If i=0.8 and r=0.6, find f. Then: apply negative, then apply contrast stretching if the only two pixel values in the image are 0.48 and 0.30 (normalised). Report the final output in 8-bit.
Topic: Multi-step transformation pipeline · Slides 16, 30, 40
f=0.48→122; neg→133; stretch: 0.48→0, 0.30→255. Final values: 133→0, 78→255
f=0.48; neg→0.52; no stretch needed. Final: 133
f=0.48→122; neg=0.52→133; stretch: min=0.48,max=0.72 → final 0,255
