01

Uninformed search — the blind algorithms

blind

Only the problem definition is visible: initial state, goal, transitions, costs. No sense of which unexplored state looks promising.

The instructor's words. BFS is "cautious" — it finishes an entire level before going one step deeper. DFS is "aggressive" — it grabs one path and rides it to the end, backtracking only once stuck.

02

Uniform Cost Search

blind + cost

f(n) = g(n) only. BFS with its queue replaced by a priority queue sorted on accumulated cost — always expand the cheapest path so far.

The rule that decides this trace. The goal is tested on expansion, never on generation. A goal can be generated looking cheap and still get beaten by a better path discovered afterward — watch it happen below.
Romania map · Sibiu → Bucharest Step 0 of 5

Optimal path: Sibiu → Rimnicu Vilcea → Pitesti → Bucharest = 278. Fagaras generated Bucharest at 310 first — it got overwritten the instant Pitesti found 278. That overwrite is the entire reason UCS waits to test the goal until expansion.
UCS pseudocode
frontier = priority_queue()   # sorted by path cost
frontier.add(start, cost=0)
explored = set()

while frontier is not empty:
    node = frontier.pop_min()      # cheapest node
    if node == goal: return path   # tested on EXPANSION
    if node in explored: continue
    explored.add(node)
    for each child of node:
        frontier.add(child, cost=g(node) + edge_cost)

From blind to informed — what is h(n)?

informed

An estimate of remaining distance to the goal, supplied from outside the raw problem definition.

The city-guide analogy. Uninformed search tries every street. Informed search asks a local: "the station is roughly 3km east." Not exact — but it slashes the search space. Something is better than nothing.
Type of h(n)What it isExampleQuality
Straight-line distanceEuclidean to goalh(Arad)=366Admissible
Manhattan distanceHoriz + vert moves8-puzzle grid movesAdmissible (grids)
Misplaced tilesCount out of place8-puzzle tile countAdmissible, weak
Learned heuristicML predicts costGame-playing agentsVaries — next lecture

Good heuristics often come from relaxing the problem: drop a constraint, solve the easier version, and its cost is automatically an admissible underestimate of the real one.

05

Greedy Best-First Search

f(n) = h(n)

Trusts the heuristic alone. Cost paid so far is invisible to it. Fast — and easily betrayed.

Romania map · h(n) only Step 0 of 3

GBFS reaches Bucharest via Arad → Sibiu → Fagaras → Bucharest = 450km — not the optimal 418. It was lured by Fagaras's low h(n) and never reconsidered the real road cost.
GBFS pseudocode
f(n) = h(n)   # ONLY the heuristic, no actual cost

frontier = priority_queue sorted by h(n)
frontier.add(start)

while frontier is not empty:
    node = frontier.pop_min_h()   # smallest h(n)
    if node == goal: return path
    for each child of node:
        frontier.add(child, priority=h(child))
06

A* Search

f(n) = g(n) + h(n)
f(n) = g(n) + h(n)g = actual cost from start  ·  h = estimated cost to goal  ·  g(start)=0  ·  h(goal)=0
Romania map · full trace Step 0 of 5

The trap. At one point Pitesti shows f=415, the lowest value among newly generated nodes. But Fagaras (f=417) was sitting in the open list from an earlier step. A* scans the entire open list before picking — never only the newest arrivals. Miss this and every A* trace in the exam goes wrong.
Optimal path: Arad → Sibiu → Rimnicu Vilcea → Pitesti → Bucharest = 418km.
A* pseudocode
open_list = priority_queue sorted by f(n)
open_list.add(start, f=h(start))

while open_list is not empty:
    node = open_list.pop_min_f()   # lowest f(n)
    if node == goal: return path
    for each successor n' of node:
        new_g = g(node) + cost(node, n')
        new_f = new_g + h(n')
        if n' not in open_list or new_f < f(n'):
            open_list.add(n', f=new_f, g=new_g)

Admissibility & consistency — interactive checker

exercise
h(n) ≤ h*(n)admissible — never overestimates the true cost-to-goal
h(n) ≤ cost(n,n′) + h(n′)consistent — triangle inequality, h can't drop faster than the real step cost
Consistent ⟹ admissible, given h(goal)=0. Not every admissible heuristic is consistent. Most heuristics in this course — straight-line, Manhattan — satisfy both.
Consistent — worked example

h(n)=10, cost(n→n')=3, h(n')=8

10 ≤ 3+8 = 11 — consistent

Not consistent — worked example

h(n)=10, cost(n→n')=2, h(n')=5

10 > 2+5 = 7 — not consistent

A worked version

Goal = node 3. h(1)=80, h(2)=60, h(3)=0, h(4)=200, h(5)=190. Defaults shown are the worked answers.

Node 2 (h=60) overestimates its true cost → inadmissible. Arc 5→1 also violates the consistency triangle inequality. One heuristic, two separate failures — admissibility and consistency are checked node-by-node and arc-by-arc, never assumed wholesale.

Exercise 1
Problem formulation — plan a trip with a delivery drone

(a) Formulate as a search problem: initial state, goal, possible actions, transition model, path cost.
(b) What is the state space? Draw a small example with 4 delivery locations.
(c) Which search algorithm would you use, and why?
(d) Write a PEAS specification for this drone agent.

Hint: drones fly in any direction but have battery limits. What does "path cost" mean for a drone?
Exercise 2
Trace A* on this graph

Nodes: S, A, B, C, D, G. Start = S, goal = G.
Edges: S→A(3), S→B(6), A→C(2), A→D(4), B→D(1), C→G(5), D→G(2)
Heuristics: h(S)=7, h(A)=5, h(B)=4, h(C)=3, h(D)=2, h(G)=0
For each expansion: show the open list, the node chosen, its f(n)=g(n)+h(n), and the final path. Then check admissibility.

Hint: always compare ALL nodes in the open list, not just recently generated ones.
Step 1: Expand S. f(S)=0+7=7. Generate A: g=3, h=5, f=8. Generate B: g=6, h=4, f=10. Open: [A 8, B 10]
Step 2: Expand A (f=8). Generate C: g=5, h=3, f=8. Generate D: g=7, h=2, f=9. Open: [C 8, D 9, B 10]
Step 3: Expand C (f=8, tied-lowest). Generate G: g=10, h=0, f=10. Open: [D 9, B 10, G 10]
Step 4: Expand D (f=9, lowest). Generate G via D: g=9, f=9 — replaces the f=10 entry found via C. Open: [B 10, G 9]
Step 5: Expand G (f=9, lowest). Goal reached. Path: S→A→D→G, cost 9.
Admissibility: h(S)=7 vs the true cheapest cost from S (9, via either A-D-G or B-D-G). 7 ≤ 9 ✓. Spot-check the rest the same way — this heuristic is admissible.
Exercise 3
Formulate clinic appointment scheduling as a search problem

(a) What is the initial state? What does the goal state look like?
(b) What are the possible actions?
(c) What is a sensible path cost function?
(d) Suggest a heuristic h(n). Is it admissible?
(e) What type of environment is this (the 6 dimensions from Lecture 2)?

Hint: multiple doctors, varied appointment lengths, patient priorities. The goal is a valid schedule with no conflicts.
Exercise 4
Admissibility check

Nodes: 1→G actual=6, 2→G actual=4, 3→G actual=2, 4→G actual=5
Heuristic values: h(1)=5, h(2)=6, h(3)=1, h(4)=4
For each node: is the heuristic admissible? Which nodes violate it? Would A* still be optimal on this graph?

Hint: h(n) admissible means h(n) ≤ true cost to goal. Check each node individually.
