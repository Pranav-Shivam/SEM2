Path Search vs Local Search
The fundamental shift: stop tracking the journey, just improve the destination

Sessions 2–4 focused on finding a path to a goal — the sequence of moves matters. Local search is different: the path to the solution is irrelevant. The state itself is the solution. For N-Queens, you only care about the final board, not which moves placed the queens there.

Fig 1 — Path search tracks every step and costs O(bᵈ) memory. Local search only holds the current state: O(1).

💡
Why it matters: For N-Queens, you don’t care how queens got to their positions — only whether they attack each other right now. Local search exploits this: start with all queens placed randomly, then keep nudging them until nobody attacks anyone.

Optimization Models — The Algorithm Family
Single-instance vs population-based; where every CS#5 algorithm belongs

Local search splits into two families. Single-instance algorithms keep one state and improve it (like one person solving a puzzle). Population-based algorithms maintain many candidate solutions simultaneously (like a team evolving together).

State Representation & Neighbours
How we encode a board and define “one step away”

A 4-Queens board state is encoded as a vector [V1, V2, V3, V4] where each value is the row of the queen in that column. Columns are fixed; only rows change.

Fig 3 — State [2,4,2,2]: V3 and V4 share row 2 (red = conflict). 1-NN gives 12 neighbouring states.

1-NN moves enumerated (from slide)

FROM SLIDESState: [V1=2, V2=4, V3=2, V4=2] 1-NN (change ONE queen): V1: 2→1 | 2→3 | 2→4 (3 moves) V2: 4→1 | 4→2 | 4→3 (3 moves) V3: 2→1 | 2→3 | 2→4 (3 moves) V4: 2→1 | 2→3 | 2→4 (3 moves) ━━━━━━━━━━━ Total 1-NN = 4 × 3 = 12 neighbours 2-NN (change TWO queens simultaneously): V1&V2, V1&V3, V1&V4, V2&V3, V2&V4, V3&V4 → 6 pairs × 3² = 54 combinations
⚠
Don’t confuse with ML’s KNN! In local search, “1-NN” means change 1 queen per step. In machine learning, KNN is a classification algorithm. Same abbreviation, completely different concept. Exams have tested this distinction.

Hill Climbing — The Core Algorithm
Always move to the best neighbour; stop when you can’t improve

Imagine climbing a hill blindfolded. Your only rule: always step in the steepest uphill direction. That’s hill climbing. Simple, fast — but you might end up on a small hill, not the highest mountain.

Exact AIMA pseudocode (from slide)

PAGE 15function HILL-CLIMBING(problem) returns a state that is a local maximum current ← problem.INITIAL while true do neighbour ← a highest-valued successor state of current if VALUE(neighbour) ≤ VALUE(current) then return current ← no improvement: local max reached current ← neighbour ← climb! VALUE(neighbour) ≤ VALUE(current) then return current current ← neighbour

HC Steps (from slide)

1Select a random state as starting point
2Evaluate the fitness scores for all successors (1-NN neighbours)
3Move to the highest-valued successor. If no successor improves, return current (local maximum)

⚡
The fatal flaw: HC only guarantees a local maximum. It gets trapped in three ways: (1) a local peak where all neighbours are worse, (2) a flat plateau where all neighbours are equal, or (3) a narrow ridge where any sideways step goes downhill. The starting state determines everything.

Hill Climbing with Random Restart
If stuck, start fresh from a completely new random state

The fix is simple: if hill climbing gets stuck, throw everything away and start from a new random state. Repeat K times and keep the best result found across all runs.

Three termination conditions (from slide, Page 18)

PAGE 18Condition 1: When no successor has a better fitness value (stuck at local optimum) → restart Condition 2: When we attain the Global Maxima or Global Minima → STOP (goal reached) Condition 3: When a predefined value of k (number of restarts) is reached → STOP (return best found)

Exact slide example (Pages 16–17)

RunStart statef at startBest successorf achievedResult
1[3, 4, 4, 2]3[3, 3, 4, 2]4Stuck → restart
2[3, 3, 4, 2]4[3, 1, 4, 2]6GOAL found! Stop.
💡
Analogy: Finding the highest mountain blindfolded. If you always start in the same forest, you’ll always climb the same small hill. Teleport to a random spot each restart — eventually you’ll land near Everest.

Stochastic HC & Simulated Annealing
Escape local optima by occasionally accepting a worse move

Instead of always picking the single best neighbour, Stochastic HC picks a neighbour weighted by probability proportional to its fitness frequency. The most common fitness level wins.

Exact slide algorithm (Page 20)

PAGE 20next ← a randomly selected successor of current ← NOT the best; random pick ΔE ← next.VALUE − current.VALUE if ΔE > 0 then current ← next ← better: always accept else current ← next with probability e(ΔE/T) ← worse: accept sometimes

Basic HC vs Stochastic HC: Basic HC always picks f=4 (the single best). Stochastic HC picks from the f=2 group (most common). Stochastic HC can intentionally move to a worse state — this is exploration, not a bug.

Simulated Annealing extends Stochastic HC: it accepts worse moves with a probability that decreases over time as temperature T cools. Inspired by slowly cooling metal so atoms settle into a perfect crystal (global optimum).

SIMULATED ANNEALINGnext ← randomly selected neighbour of current ΔE ← FITNESS(next) − FITNESS(current) if ΔE > 0: current ← next ← better move: always accept else: current ← next with P = e(ΔE/T) ← worse move: accept with prob where T = temperature (starts high, cools each step) As T → 0: P → 0 (cold = almost no bad moves accepted) As T → ∞: P → 1 (hot = almost everything accepted = random walk)

Local Beam Search
K parallel hill climbers that share their best discoveries every round

Send K agents simultaneously. At every step, all K agents pool their successors together and select the globally best K to continue. This is the critical difference from K independent restarts.

Exact algorithm steps (from slide)

PAGE 22Step 1: Initialize k random states Step 2: Evaluate fitness scores for all successors of all k states Step 3: If goal found → STOP Else: select best-scoring k successors (from ALL successors combined) Step 4: Repeat from Step 2

Exact slide example k=2 (Page 22)

BeamStateBest successor foundh(n)Result
Beam 1[3,4,4,2][3,3,4,2]4Continue
Beam 2[3,4,4,2][3,1,4,2]6GOAL — all beams stop
⚠
K restarts vs Beam Search — the critical difference:
K restarts: Each run is completely isolated. Run 1 finishes, then Run 2 starts. No communication.
Beam Search: All k states run together. At each step, successors from ALL k states are pooled and the globally best k are kept. States cooperate and concentrate resources on the most promising region.

Stochastic Beam Search
Add probability to beam search to prevent all beams converging to the same local optimum

Regular Beam Search always picks the top-k by fitness (greedy). Stochastic Beam Search selects the next k states proportionally to fitness (roulette-style), maintaining diversity and avoiding premature convergence.

Exact steps from slide (Page 24)

PAGE 24Step 1: Initialize k random states Step 2: Evaluate fitness scores for all successors of k states Step 3: Calculate the probability of selecting a successor based on fitness score Step 4: Select the next state based on the highest probability Step 5: If goal not found, select next k states randomly based on probability (not just top-k) Step 6: Repeat from Step 2
AlgorithmSuccessor selectionRisk of all beams converging
K Random RestartsEach isolated, keeps own bestHigh (independent runs)
Local Beam SearchGreedy top-k from all combinedMedium (can converge to same region)
Stochastic Beam SearchProbabilistic by fitness frequencyLower (diversity maintained)

Genetic Algorithms — Theory & Vocabulary
Evolution borrowed from biology: populations, fitness, selection, crossover, mutation

From Page 28 (Basic GA):

Start with a large population of randomly generated solutions. Repeatedly: evaluate each solution, (probabilistically) keep a subset of the best solutions, use these to generate a new population. Quit when you have a satisfactory solution or run out of time.

A Genetic Algorithm (GA) is a global search heuristic that solves optimisation problems by mimicking biological evolution. Instead of one agent climbing a hill, a whole population of candidate solutions evolves over generations toward better answers.

🧬
From the slides (Page 25): GAs use techniques inspired by evolutionary biology — Inheritance, Mutation, Selection, Crossover. Evolution starts from a population of randomly generated individuals and happens in generations. Each generation: evaluate fitness → select fittest → modify to form next generation.

8-Queens Problem — Why GA fits

Place 8 queens on an 8×8 chessboard so no queen attacks any other. Search space = 8&sup8; = 16,777,216 possible placements. HC alone often fails. GA excels by combining good “left side” from one board with good “right side” from another via crossover.

💡
Chromosome for 8-Queens (Page 29): Each individual is represented as a sequence of 8 integers: [q1, q2, q3, q4, q5, q6, q7, q8]. Each value = row of the queen in that column. Example: [1, 5, 3, 6, 4, 2, 8, 7] means Q1→row1, Q2→row5, Q3→row3, Q4→row6, etc.

GA key techniques (Page 45)

1Design of the fitness function — must correctly score what you want to optimise
2Ensuring diversity in the population — without it, all individuals converge to the same local optimum
3Randomization — for selection (roulette), crossover point choice, and mutation

Ant Colony Optimization (ACO)
Pheromone trails + evaporation = emergent collective intelligence

Real ants find the shortest path to food without any central planner. Each ant deposits a pheromone trail; shorter paths are traversed more often, accumulating more pheromone. Other ants prefer high-pheromone paths. Pheromone evaporates, so bad paths fade. The colony collectively solves the shortest-path problem.

ACO Pseudocode (Page 48)

PAGE 48Procedure ACO Schedule Activities: Initialization: Place N ants on starting node Set α, β, ρ parameter values Set τ₀ = 0 ← initial pheromone = zero on all edges Construction: Each ant builds a tour using the NTP formula to choose next node Update Pheromone: Evaporate: τ(ij)ₘₑ⁗ = (1-ρ) × τ(ij)ₒₑₓ Deposit: τ(ij) += Δτ(ij)ₖ = Q / fₖ (if ant k used edge i→j) Daemon Actions {optional}: Local search, elitism (extra pheromone for best-ever solution) End Schedule Activities End ACO

Exercise 1 — Slide Page 52Local Beam Search — Which 3 States Win?
▼
k=3 current states: S1=12, S2=15, S3=10. Their successors: A=16, B=11, C=14, D=18, E=13, F=17. Goal: maximise. Which 3 states are selected for the next beam?
Solution
Key rule: Beam Search selects top-k from ALL successors combined (not per-state best). All 6 successors with exact heuristic values: A = 16, B = 11, C = 14, D = 18 ← highest E = 13, F = 17 ← second Ranked by fitness: D (18) > F (17) > A (16) > C (14) > E (13) > B (11) ← #1 selected F = 17 ← #2 selected A = 16 ← #3 selected C = 14 E = 13 B = 11 Answer: D, F, A are the next k=3 beam states. Current states S1=12, S2=15, S3=10 are REPLACED entirely. This is the key difference from restarts: all successors pooled globally.

Exercise 2b — Page 53Roulette Wheel Selection
▼
Individuals and fitness: A | 10, B | 30, C | 40, D | 20. Total = 100. Which has the highest selection probability? Why?
Solution
Total fitness = 10+30+40+20 = 100 P(A) = 10/100 = 10% P(B) = 30/100 = 30% P(C) = 40/100 = 40% ← HIGHEST P(D) = 20/100 = 20% Total = 100% ✓ Individual C has the highest selection probability (40%) because it has the highest fitness. Even A at 10% has a real chance — this maintains genetic diversity in the population.
Exercise 2c — Page 54Relaxed Problems & Admissible Heuristics
▼
A relaxed problem is obtained by removing one or more constraints from the original problem. Consider these relaxations of the warehouse robot problem. Consider these relaxations of the warehouse robot problem. Answer: which gives weakest/strongest heuristic? Why are relaxed-problem costs admissible?
Solution
Definition: A relaxed problem removes one or more constraints from the original. Its solution cost is always ≤ the original cost. Relaxations given: R1: Ignore all locked-door constraints (robot moves freely) R2: Ignore package collection order (shortest distance to exit) R3: Robot collects any package instantly in same connected component R4: Ignore robot’s current location; estimate min cost to connect packages+exit Weakest heuristic: R1 (removes most constraints → solution costs least → estimate is lowest) Strongest heuristic: R4 (removes fewest useful constraints → still close to real cost) Why admissible? Removing constraints makes the problem easier than the original. Optimal cost in relaxed problem ≤ optimal cost in original problem. Therefore: h(n) = relaxed cost never overestimates the real cost → admissible.
