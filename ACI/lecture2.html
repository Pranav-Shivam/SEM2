3. Rational Agents and Ideal Rational Agents

TermDefinitionKey insight
AgentPerceives + actsNo mention of "right thing" — just perceiving/acting
Rational AgentStrives to do the right thing — the action that maximises performanceCorrect action ≠ correct outcome every time
Ideal Rational AgentFor every percept sequence, takes the action that maximises expected performance given built-in knowledge + evidence so farJudged on what was known then, not in hindsight
💡 Rationality ≠ Omniscience — the retrospection principle Omniscience = knowing everything in advance. Rationality only needs the best decision given current knowledge. "You could've done better" is not a valid criticism if the agent used all available evidence optimally at the time. A rational agent can still produce a bad outcome in hindsight — that alone doesn't make it irrational.

5. Six Environment Dimensions EXAM

Click each bar below — drag your eye left→right. The harder end (right) is where the real world sits, on every axis at once.

Fig 4 — Six independent axes. An environment doesn't have to pick the same side on every axis.
DimensionMeaningExamples
Fully vs Partially ObservableDo sensors give the complete state at every step?Chess, tic-tac-toe (full) · Poker, autonomous driving (partial)
Deterministic vs StochasticIs the next state fixed by current state + action alone?Calculator (det.) · weather, football, poker (stoch.)
Episodic vs SequentialAre decisions independent episodes, or does the current choice affect future ones?Spam filter, "is there a cat in this image?" (episodic) · chess, trading bot (sequential)
Static vs DynamicCan the environment change while the agent is still deciding?Crossword, Rubik's cube (static) · driving (dynamic) · chess-with-a-clock = semi-dynamic
Discrete vs ContinuousFinite, countable percepts/states/actions, or infinite?Chess moves (discrete) · taxi speed/steering angle (continuous)
Single vs Multi-AgentAre other acting entities present (cooperative, competitive, or indifferent)?Crossword (single) · chess (competitive multi) · RoboCup (multi)
⚠ Exam point — autonomous driving is PARTIAL, not full Cameras all around still can't see what's 2 km ahead, when a signal will change, or what the car behind intends. Chess: you always see the whole board. That contrast is the definition of full vs partial observability.
⚠ Common mistake — chess is DETERMINISTIC Determinism is about whether the same action gives the same result, not about whether the game has uncertainty in outcome. Moving a bishop diagonally always does the same thing. Compare a dice roll: same action, different number → stochastic. Football: kicking the ball can go many ways because other players interfere → stochastic.
⚠ The "semi-dynamic" trap — chess with a clock The board itself doesn't change while you think (so it's static, by the board alone). But your score — remaining time on the clock — keeps ticking whether you move or not. That in-between case is called semi-dynamic: the environment doesn't change, but the agent's performance measure does. Expect this exact example if the exam asks for something other than plain static/dynamic.

6. Five Agent Types — what's behind the "?"

Each type below extends the one above it — same building, one more floor.

Fig 5 — Five architectures, increasing sophistication top to bottom.
Anatomy of a learning agent (4 parts) Performance element — chooses actions (this is "the agent" from types 1–4).
Learning element — improves the performance element.
Critic — scores actions against a fixed performance standard (e.g. "−2 reward").
Problem generator — deliberately suggests exploratory (sub-optimal) actions so better behaviour can be discovered.
🚦 The brake example — a full learning loop Percept: red signal at 15 m. Action: apply 17 kN brake. Result: stopped 2 m past the zebra crossing. Critique: negative. Learning: next time at 15 m, apply 15 kN instead.
Act → Observe → Critique → Update → do better next time. This loop is literally the seed of reinforcement learning.

8. Problem-Solving Agent → Search Formulation

Definition A Problem Solving Agent (PSA) is a goal-based agent that turns its goal into a search problem: (1) Formulate — define the 5 components → (2) Search — find the action sequence → (3) Execute — carry it out. This only works cleanly when the environment is observable, discrete, known, and deterministic.
Fig 7 — 5-component pipeline + a worked state-space graph. Solution = any path to goal; optimal = cheapest path.

Abstraction Real states have endless irrelevant detail (you could turn the radio on while driving Arad→Sibiu). A good problem formulation deliberately throws away anything that doesn't matter for reaching the goal.

8b. States, Nodes & the Frontier EXAM

Three words that look interchangeable but mean different things — a favourite place to lose easy marks.

State vs Node — not the same thing State = a configuration of the world (e.g. "In(Sibiu)" or a specific vacuum-room layout). The state space is the graph of all states reachable from the initial state — nodes = states, arcs = actions.

Node = a bookkeeping data structure used by the search algorithm, not part of the world itself. Every node stores: n.STATE, n.PARENT, n.ACTION, n.PATH-COST g(n)

Because a node carries a path, the same state can be wrapped in two different nodes if two different paths reach it — one node per path, not per state.
Fig 7b — Why "state" and "node" are not synonyms.
The Frontier The frontier = the set of nodes that have been generated but not yet expanded ("the edge of explored territory"). How you store the frontier is the search strategy:
  • FIFO queue (first-in-first-out) → Breadth-First Search
  • LIFO stack (last-in-first-out) → Depth-First Search
  • Priority queue (lowest cost / lowest f(n) first) → UCS / A*
Fig 7c — Explored / Frontier / Unexplored. The frontier is the only part the algorithm actively chooses from.

8c. Evaluating a Search Strategy — 4 Criteria

Every algorithm in the next table is judged on exactly these four axes.

CriterionQuestion it answers
CompletenessDoes it always find a solution when one exists?
OptimalityDoes it always find the least-cost solution?
Time complexityHow many nodes get generated?
Space complexityMax nodes held in memory at once?
Notation used everywhere in Ch.3 b = branching factor (children per node) · d = depth of the shallowest goal · m = maximum depth of the whole state space (can be ∞ for an infinite tree).

12. Worked Numericals

① How many states in the vacuum world?

Two locations, each independently Clean or Dirty, plus where the agent is.

locations the agent can be in: 2
dirt status of room A: 2 (Clean / Dirty)
dirt status of room B: 2 (Clean / Dirty)
total states = 2 × 2 × 2 = 2 × 2² = 8

Matches Fig 6 above — 8 nodes drawn, exactly 2 of them green (goal states).

② How many reachable states in the 8-puzzle?

9 cells, 9 distinct tiles (8 numbered + 1 blank) → naive arrangements = 9!.

naive permutations = 9! = 362,880
BUT: sliding-tile moves preserve permutation parity
only half of all 9! arrangements are reachable from any given start
reachable states = 9! / 2 = 362,880 / 2 = 181,440

Exam trap: if asked "total states" they may want 362,880 (raw); if asked "reachable from a given start," it's 181,440. Read the question carefully.

③ Optimal path cost in the state-space graph (Fig 7)

Edges: Start→S₁ = 10, Start→S₂ = 15, S₁→S₃ = 8, S₁→S₄ = 12, S₂→S₃ = 6 (not all drawn), S₂→S₄ = 6, S₃→Goal = 9, S₄→Goal = 14.

Path A: Start→S₁→S₃→Goal = 10 + 8 + 9 = 27
Path B: Start→S₁→S₄→Goal = 10 + 12 + 14 = 36
Path C: Start→S₂→S₄→Goal = 15 + 6 + 14 = 35
minimum = Path A = 27 ✓ optimal

This is exactly what UCS / Dijkstra would return — it always expands the cheapest frontier node first, so it never commits to Path B or C before checking A.

④ Branching factor & depth — quick BFS memory estimate

If branching factor b = 3 and the shallowest goal is at depth d = 4, roughly how many nodes might BFS generate in the worst case?

worst case nodes ≈ b⁰ + b¹ + b² + b³ + b⁴
= 1 + 3 + 9 + 27 + 81
= 121 nodes

This is why BFS is "complete but exponential" — b and d both push the count up fast. With b=10, d=4 this would already be ~11,111 nodes.

14. Extra Practice (worked answers to Exercises 1 & 5, plus bonus questions)

Q. Is "playing poker online against 3 strangers" episodic or sequential? Justify.

Sequential. Your bet now changes the pot, the other players' beliefs about your hand, and your own chip stack for every future round in the same game. An episodic task (like classifying one image) has no such carry-over between instances.

Q. A self-driving car's environment — is it semi-dynamic, dynamic, or static? Why does this matter for design?

Dynamic. Pedestrians, other vehicles, and signals change state while the car is still "thinking" (planning its next move). This forces the agent design to budget a hard real-time deadline on its decision cycle — unlike a static environment (e.g. solving a crossword) where the agent can think as long as it wants.

Q. Give one example each of a goal-based and a utility-based agent doing the same task, and explain the difference in behaviour.

Task: route planning. Goal-based agent: any path that reaches the destination is equally acceptable — it stops at the first valid plan satisfying the goal test. Utility-based agent: among many valid routes, it ranks them by a utility function (e.g. 0.6×time + 0.4×comfort) and picks the highest-scoring one — it can express preference, not just success/failure.

Exercise 1 worked answer — PEAS for a satellite image analysis system (cloud cover matters)

P: classification accuracy, false-positive rate, processing latency, cost per image.
E: incoming satellite image feed, atmospheric conditions, ground-truth labels (when available).
A: tags/labels output, alert triggers, database writes.
S: raw pixel sensor data, metadata (time, location, satellite orientation), cloud-cover flags.

Observability: partial — cloud cover can hide the true ground state from the sensor at that moment, even though the satellite itself can "see" everything optically.
Determinism: stochastic — atmospheric noise and sensor error mean the same ground truth doesn't always produce the same image.

Exercise 5 worked answer — RoboCup robot football, agent type recommendation

Environment: multi-agent, stochastic, partially observable, dynamic, continuous, sequential — about as hard as it gets on every axis.

Best-fit architecture: utility-based (often layered with learning). A simple reflex agent can't plan a pass; a pure goal-based agent treats "score a goal" as binary and can't trade off between several valid options (dribble vs pass vs shoot). A utility function lets it weigh risk of losing the ball vs probability of scoring, and a learning loop lets it adapt its utility estimates from match experience (this is exactly how real RoboCup teams use RL).
